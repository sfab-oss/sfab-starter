# Internationalization (i18n)

App UI copy is catalog-first with **Paraglide JS**. English is the source of
truth; Spanish ships as the first target locale. Locale preference is a
**cookie** (`PARAGLIDE_LOCALE`) — no `/es` URL prefixes and no DB column in v1.

## Layout

```
packages/i18n/
  messages/en.json          # SOURCE OF TRUTH
  messages/es.json          # Spanish target (agent/human maintained)
  glossary/es.md            # tone + preferred terms for ES
  .i18n-lock.json           # EN value hashes (incremental retranslate)
  project.inlang/           # Paraglide / inlang settings (en + es)
  scripts/sync.ts
  scripts/lint.ts

apps/web/
  src/paraglide/            # generated by Vite / `pnpm i18n:compile` (gitignored)
  src/components/common/language-switcher.tsx
```

Runtime messages are compiled into `apps/web/src/paraglide`. Components import:

```ts
import { m } from "@/paraglide/messages.js";
import { getLocale, setLocale } from "@/paraglide/runtime.js";

m.nav_home(); // "Home" | "Hoy"
```

Money / dates: pass `intlLocale()` from `apps/web/src/lib/locale.ts` into
`formatMoneyMinor(..., { locale })` and `date-fns` locales.

Registry block copy is **out of scope** (follow-up ALW-571).

## Commands

Run from the monorepo root:

| Command | Purpose |
|---|---|
| `pnpm i18n:sync` | Ensure every EN key exists in each target (empty stub if missing); refresh lock hashes for filled keys |
| `pnpm i18n:lint` | Fail on missing/empty targets or `{placeholder}` mismatch |
| `pnpm i18n:lint -- --json` | Machine-readable `{ gaps, warnings }` for agents |
| `pnpm i18n:compile` | Compile Paraglide into `apps/web/src/paraglide` (also runs before `web` typecheck) |

`i18n:lint` is wired into CI next to `lint:check`.

### Incremental / hash lockfile

`.i18n-lock.json` stores a SHA-256 of each EN string the last time a non-empty
target existed. After you change English copy:

1. `pnpm i18n:sync` — refreshes hashes for keys that still have target text
2. `pnpm i18n:lint` — warns (`stale_en_hash`) when EN changed but the target
   was not revisited; fix by updating `es.json` (or other locales) then sync

There is **no required LLM API key**. Optional `i18n:translate` may appear
later; until then agents fill targets from the lint gap list.

## Agent workflow A — add a locale (e.g. French)

1. Add `"fr"` to `packages/i18n/project.inlang/settings.json` `locales`
2. Add `"fr"` to `packages/i18n/src/locales.ts` (`LOCALES`, labels, `INTL_LOCALE`)
3. Create `packages/i18n/messages/fr.json` as `{}`
4. Optional: `packages/i18n/glossary/fr.md`
5. `pnpm i18n:sync` → stubs every EN key into `fr.json`
6. `pnpm i18n:lint -- --json` → list gaps
7. Fill `fr.json` (agent edits JSON; no vendor key required)
8. `pnpm i18n:lint` green
9. `pnpm i18n:compile` — language switcher reads `LOCALES` automatically

## Agent workflow B — develop as you go

1. Add keys to `messages/en.json` first (never hardcode user-visible UI strings)
2. Use `m.your_key()` in components
3. `pnpm i18n:sync` → stub targets
4. Fill new target strings (from lint `--json` if helpful)
5. `pnpm i18n:lint` green, then typecheck / test

Anti-pattern: shipping `"Void invoice"` as a JSX literal. Lint cannot see it.

## Strategy notes

Paraglide strategy is `["cookie", "baseLocale"]` — default remains **en**. We
intentionally omit `preferredLanguage` so browser Spanish does not silently
override English users.
