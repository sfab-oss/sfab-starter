# ALW-554 README getting started — agent-first, uncluttered

## Context

Post-launch README still leads with a human bash-block quickstart (seed credentials, comments inside the code fence) and keeps "Hand it to an AI agent" as a separate later section. In 2026 the primary reader is an AI coding agent; humans and agents should share one instruction path.

## Scope

- Rewrite README **Getting started** so agent handoff is the first option, then shared run steps for human or agent.
- Move prose out of code fences (commands only inside fences).
- Demote `pnpm db:seed` / demo credentials out of the main path (short doc link is fine).
- Remove the duplicate standalone "Hand it to an AI agent" section once Getting started owns that path.
- Keep stack / what's included / layout unless a one-line cross-link is needed.

## Acceptance criteria

- [ ] AC-1: Getting started opens with an agent handoff option before local run steps. verify: README.md
- [ ] AC-2: Code fences contain commands only; env/seed/sign-in guidance is outside fences. verify: README.md
- [ ] AC-3: Seed + demo credentials are not required in the main path; documented in a linked file if kept. verify: README + linked doc
- [ ] AC-4: No separate top-level "Hand it to an AI agent" section duplicating Getting started. verify: README.md headings

## Non-goals

- Changing seed script behaviour; factory scrub of `.sfab/`; launch copy.

## Boundaries

- ✅ Always: same steps for human and agent; link `docs/template-init.md` and `AGENTS.md`.
- ⚠️ Ask first: renaming or deleting `pnpm db:seed`.
- 🚫 Never: commit `.dev.vars`; invent credentials not in the seed script.
