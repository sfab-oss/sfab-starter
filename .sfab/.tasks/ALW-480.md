# ALW-480 Verify destructive-approval path on main (post-ALW-456)

## Context

[[ALW-456]] merged the codemode mid-script approval path for `delete-product` and was meant to supersede [[ALW-454]] (top-level Approve lost tool-call ID). Before public launch we must confirm the **destructive approval** happy path on current `main` — this is a hard launch gate for an AI-native ERP template.

## Scope

- On `origin/main` (or a fresh worktree): trigger delete-product approval via the shipped path (codemode / pending execution UI from ALW-456).
- Exercise Approve, Reject, and reload-mid-pause.
- If green: cancel [[ALW-454]] (if still open) and note evidence on [[ALW-427]].
- If red: reopen/narrow a fix task with the failing cell; do not launch until fixed.

## Acceptance criteria

- [ ] AC-1: Approve completes without tool-call-ID error; disposable product is deleted. verify: browser + catalog
- [ ] AC-2: Reject denies and leaves the product intact. verify: browser
- [ ] AC-3: Reload while awaiting approval still shows Approve/Reject and both still work. verify: browser reload
- [ ] AC-4: Evidence screenshots/notes posted; [[ALW-454]] dispositioned (canceled if superseded, or fix filed if still broken)

## Non-goals

- Pretty approval-card labels ([[ALW-401]]); gating create/update-product; attachment/page-context fixes.

## Boundaries

- ✅ Always: disposable local products only; seed SKUs untouched.
- 🚫 Never: prod keys; delete seed catalog products.
