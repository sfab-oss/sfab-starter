# ALW-421 Catalog: migrate list to ResourceTable (resource-list-page filters)

## Context

Dogfood QA (B-7) flagged the live Catalog list as the legacy `DataTable` (single search + Columns toggle). The ideal pattern already exists in the starter registry: `resource-list-page` + `ResourceTable` with `filterDefinitions` / filter toolbar, no Columns toggle by default, optional row actions.

ALW-419 shell polish is merged; this is the follow-up for Catalog only.

## Scope

- Migrate `apps/web` Catalog list (`/_protected/catalog/`) from legacy `DataTable` to `ResourceTable`.
- Wire filter toolbar using `filterDefinitions` aligned with what the products list API already supports (at least name/SKU text search; add more only if the API already exposes them).
- Disable Columns visibility toggle (`showColumnVisibility={false}` / ResourceTable default path).
- Keep compact header **Add Product** CTA; keep product name link to detail.
- Prefer URL search-param sync for pagination/sort/filters consistent with current catalog route.

## Acceptance criteria

- [ ] AC-1: Catalog list uses `ResourceTable` (not legacy search+Columns chrome). verify: Catalog page screenshot / DOM
- [ ] AC-2: Filter toolbar is present with at least a text filter that searches products (name/SKU via existing API). verify: type filter → list updates
- [ ] AC-3: Columns toggle is not shown on Catalog. verify: no Columns button
- [ ] AC-4: Pagination + sort still work via URL/search params. verify: sort Name/Price; page controls
- [ ] AC-5: Empty catalog and filtered-empty states remain usable. verify: empty seed / filter with no matches

## Non-goals

- Invoice UI redesign (B-8).
- Expanding the products API with new filter dimensions unless required for AC-2.
- Full resource-list collection-state scaffolding beyond what Catalog needs.

## Boundaries

- ✅ Always: reuse `@workspace/ui` `ResourceTable` + `table-filter-types` / registry `resource-list-page` patterns.
- ⚠️ Ask first: changing products list API contract.
- 🚫 Never: redesign invoice composer in this task.
