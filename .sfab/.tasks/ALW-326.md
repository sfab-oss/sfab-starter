# ALW-326 Scaffold the knowledge layer + Tier-1 primitive skeleton (ADR-0010 §E/§H)

## Context

The template architecture is settled in **ADR-0010 (Amendment 2026-06-18)** and **ADR-0017**, and the **directory/package structure is decided in [[ALW-327]] (D1–D10)**. This task lays down the *structure* those imply — the **directory/package layout + renames**, the **knowledge layer** (canonical `docs/`, droppable `apps/docs` renderer, `.agents/skills` + runtime symlinks, `AGENTS.md`), and a **skeleton** for the Tier-1 base primitives — **without authoring content or implementing primitives**. The repo already has much of this; the job is to reshape it to match the decisions and add the missing scaffolding.

## Scope

### A. Directory & package structure (implements [[ALW-327]] D1–D10)
- **Renames (role over technology):** `apps/web-tanstack`→`apps/web`; `packages/db-d1`→`packages/db`; `packages/types`→`packages/contract`; `packages/cloudflare-env`→`packages/env`; `packages/typescript-config`→`packages/tsconfig`. Keep workspace scope `@workspace/*`. Update `cf-typegen` path coupling for the `env` rename. Remove any `primitives` package.
- **`contract` becomes a pure boundary-zod leaf:** no workspace deps; **remove the drizzle→zod derivations** currently in `db-d1/src/types/*` (the `createInsertSchema(...).omit(...)` files). Boundary zod is hand-written, feature-keyed (`contract/<cap>/`).
- **Layer model + cross-layer feature-key consistency:** `db` (schema one-file-per-capability + client + `$infer`) · `contract` (`<cap>/`) · `core` (queries + domain fns, `<cap>/`; params `z.infer<contract>`) · `apps/web` surfaces (`hono/{public,protected,org-protected}/<cap>/`, MCP, AI DOs) · `packages/agent/tools/<cap>.ts` · `apps/web/src/components/<cap>/`. Restructure `core` flat→`<cap>/` folders (e.g. `core/items.ts`+`core/locations.ts`→`core/catalog/`) so the `<cap>` key is identical across layers.
- **Type model rules:** ISO `text` timestamps everywhere for domain tables (fix the `mode:"timestamp"/"timestamp_ms"` stragglers; Better-Auth `Date` columns are an accepted internal-only one-off); money = numeric→`number` customType.
- **Mechanical boundaries (D8):** set each `package.json`'s deps to the allowed graph — `contract` (no workspace deps) · `db` (leaf) · `core` = db+contract · `ui` = contract (NOT core/db) · `apps/web` = core+contract+ui. Add a CI **cycle check** (`madge --circular` or equivalent).
- **Lint:** drop **Ultracite**; keep Biome with a repo-owned config (the concrete ruleset is a separate deferred task — wire the empty/own config here, don't author the full ruleset).

### B. Knowledge layer (ADR-0010 §H)
- **Canonical docs at repo root.** `docs/` is the single source of truth (`architecture/`, `decisions/`, `patterns/`/`template-best-practices/`). If markdown lives in `apps/docs/content/`, move it to root `docs/` and make `apps/docs` a **thin renderer** (no duplicated content).
- **Droppable docs app.** Record the rule: drop the `apps/docs` *app* at fabrication, **keep `docs/`**.
- **Skills home.** `.agents/skills/` canonical; committed `.claude/skills` + `.agent/skills` symlinks resolve to it. Add one placeholder template-architecture skill whose `SKILL.md` is a thin trigger that **points into `docs/` by plain path**.
- **AGENTS.md** as the map (docs location, skills index, read-before-act). Align the existing file.

### C. Tier-1 primitive skeleton (ADR-0010 §E)
- Lay out folders/stubs for the Tier-1 primitives where they live (db schema / contract / core / surfaces / components) — directories + minimal stubs that **typecheck**, not implementations. Just enough that the parts list is legible.
- **Manifest shape.** Ensure `.sfab/template.json` can carry the `packs` provenance field (ADR-0017 §3) — shape only, no behavior.

## Non-goals

- Authoring real docs content (architecture writeup, code-smells) — separate content tasks; the **formal `docs/architecture` doc + ADRs are gated** by the docs-structure exploration.
- Authoring real skills content — separate.
- Implementing any Tier-1 primitive (Transaction Core, activity log, i18n) — separate build tasks.
- Authoring the full Biome ruleset (only wire the repo-owned config) — separate task.
- The registry/pack mechanism behavior (`registry.json`, install skills, manifest *behavior*) — [[ALW-324]] / ADR-0017.
- De-verticalizing the example beyond the structural renames/restructure — [[ALW-299]].

## Boundaries

- ✅ Always: structure only; stubs must typecheck; match [[ALW-327]] D1–D10 + ADR-0010 §E/§H + ADR-0017 §6.
- ✅ Always: skills reference docs by path mention; symlinks only for runtime exposure of `.agents/skills`.
- ⚠️ Ask first: deleting/relocating existing docs content beyond the `docs/` move; changing how the template is tagged/released.
- 🚫 Never: author primitive implementations or real doc/skill content; reintroduce a `types`/`primitives` package or a drizzle→zod chain; couple anything to a vertical's domain naming.
