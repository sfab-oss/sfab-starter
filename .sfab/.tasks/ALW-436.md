# ALW-436 Org switcher: remove or wire ⌘1..N shortcut labels

## Launch cut (2026-07-10)

**Status: soft stretch** under [[ALW-479]]. Shell nit in scope for launch polish.

## Context

From [[ALW-423]] QA (finding S-1): the org switcher dropdown in `app-sidebar-footer.tsx` renders `DropdownMenuShortcut` labels `⌘1`, `⌘2`, … for each organization, but no keyboard handler binds those chords. Pressing ⌘1 while the menu is open does nothing. Misleading chrome on a high-traffic control.

## Scope

- Either remove the `DropdownMenuShortcut` labels until shortcuts are real, **or** wire `⌘1..N` (and a documented Windows/Linux equivalent) to `setActiveOrganization` for the Nth org in the list.
- If wiring: remount/reset org-scoped UI (chat dock) the same way a click does today.

## Acceptance criteria

- [ ] AC-1: No unbound shortcut labels appear in the org switcher (verify: open footer menu — either no ⌘N badges, or ⌘N switches org).
- [ ] AC-2: If shortcuts are wired, ⌘1 selects the first listed org and the active org indicator updates (verify: keyboard-only).

## Non-goals

- Global app-wide hotkeys beyond the org switcher.
- Creating organizations.

## Boundaries

- ✅ Always: keep click-to-switch behavior intact.
- ⚠️ Ask first: binding shortcuts that conflict with browser/OS defaults.
- 🚫 Never: change org membership or auth.
