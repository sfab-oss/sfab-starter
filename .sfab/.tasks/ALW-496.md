# ALW-496 Smell lint gates (useEffect justify) — after Ultracite exit

## Context

Under [[ALW-494]], after Ultracite is gone from starter ([[ALW-495]]) and (ideally) platform. Biome cannot distinguish the documented `useEffect`-sync smell from legitimate external sync — gate is "justify every effect" plus a baseline pass.

## Scope (draft — sharpen when unblocked)

- Inventory enforceable smells vs AI-review-only.
- First candidate: restrict `useEffect` / `useLayoutEffect` from `react` (likely `warn`, scoped away from shadcn stock UI).
- Second candidate (owner, 2026-07-22): **raw `console.*` ban in server-side code** — enforce the structured-logging helper from [[ALW-662]] via Biome's built-in `noConsole` (exception: the helper's own module; client/browser code scoped separately). [[ALW-662]] lands an interim CI guard; when this task runs, fold that guard into the owned ruleset so there's one enforcement point. Same rule ships in the starter ruleset once the starter adopts the logging convention (follow-up task, after ALW-662 settles the envelope). Rationale: mechanical convention checks belong in deterministic lint gates, not AI review — review judges only what lint can't (kind choice, severity, payload quality).
- Justify or refactor existing call sites.
- Cross-link starter guides / platform `code-smells.md`.
