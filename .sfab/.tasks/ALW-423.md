# ALW-423 QA: shell, navigation & command-K palette (desktop + mobile)

## Context

The global chrome is the highest-traffic surface: sidebar (`components/layout/app-sidebar.tsx`), org switcher / account footer (`app-sidebar-footer.tsx`), breadcrumbs, theme toggle, and the ⌘K search palette (`components/search/search-command.tsx`). The palette is known to be a stub — static 3-page list (Documents missing), hardcoded `isLoading = false`, unwired debounce — and the org switcher shows ⌘1..N shortcut labels that are not bound. First dogfood pass fixed mobile sidebar auto-close and breadcrumb doubling ([[ALW-419]]); those fixes need regression verification.

## Scope

- Sidebar: expand/collapse rail, active states, mobile sheet open/auto-close-on-nav, trigger reachability.
- Breadcrumbs on every route incl. leaf duplication regression (Settings, Today, invoice routes).
- Org switcher: switch org (chat dock must remount for the new org), create org entry, log out; verify ⌘1..N labels vs actual behavior.
- Command palette: ⌘K, Ctrl-K, and `/` open/close/toggle; guard when focus is in inputs/textareas/contenteditable; keyboard-only navigation and Enter-to-open; result coverage vs actual pages; empty state.
- Theme toggle from both variants (icon and menu row).
