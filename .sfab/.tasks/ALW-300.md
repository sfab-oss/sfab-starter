# ALW-300 Add GitHub Actions CI workflow (lint, typecheck, test, build)

## Context

`sfab-starter` has no GitHub Actions workflows. Quality gates only run locally via the Husky pre-commit hook — which can be skipped (`--no-verify`) or never run if a contributor pushes from a machine without hooks. The main `sfab` platform repo runs `lint`, `typecheck`, `test`, and `build` on every PR via `.github/workflows/ci.yml`.

As a public template others fork, the starter needs the same **remote** backstop without copying platform-only concerns (deploy workflow, Blacksmith runners, workflow-integration quarantine jobs).

## Scope

- Add `.github/workflows/ci.yml` triggered on `pull_request` (and optional `workflow_dispatch`).
- Jobs (parallel): `lint` (`pnpm lint:check`), `typecheck`, `test` (`pnpm test`), `build` (`pnpm build`).
- Use standard `ubuntu-latest` runners + `pnpm/action-setup` + `node-version: 22` + frozen lockfile install — mirror `sfab` CI shape, not its infra extras.
- Set `NODE_OPTIONS: --max-old-space-size=4096` on the `build` job (same rationale as sfab CI / ALW-127).
- Add `concurrency` group per ref with `cancel-in-progress: true`.
- Do **not** add deploy workflow, e2e, or platform-specific test splits.

## Acceptance criteria

- [ ] AC-1: `.github/workflows/ci.yml` exists and runs on `pull_request`. verify: open a PR → four jobs appear in GitHub checks.
- [ ] AC-2: All four jobs pass on `main` and on a sample PR branch. verify: green checks on the PR that adds this workflow.
- [ ] AC-3: `pnpm lint:check`, `pnpm typecheck`, `pnpm test`, and `pnpm build` are the commands each job runs (no drift from local scripts). verify: read workflow file.

## Non-goals

- Deploy workflow or Cloudflare secrets wiring.
- E2E / Playwright in CI (keep local/`test:e2e` only for now).
- Blacksmith or other custom runners.
- Changing what `pnpm test` covers (still `web-tanstack` Vitest).

## Boundaries

- ✅ Always: PR-only; keep workflow minimal; match existing root package scripts.
- ⚠️ Ask first: adding required status checks in GitHub branch protection (may need public repo / Team plan).
- 🚫 Never: hand-edit `.sfab/.tasks/*`; copy `deploy.yml` from sfab.
