# ALW-218 Bring in the Think-based organization ERP agent (OrgAgent + OrgChat DO pattern from sfab)

## Context

sfab-starter currently runs its AI agent as a stateless `AgentConfig` + `streamText` flow (`src/lib/ai/respond.ts`). The sfab platform repo has a full Think-based durable agent built on `@cloudflare/think` + the `agents` SDK â€” Durable Object classes (parent instance + Think session facets), shared workspace, codemode, multi-chat sessions, shared memory, compaction, and recovery.

This task ports that architecture into sfab-starter as an **organization-level ERP agent** (`OrgAgent` + `OrgChat`), scoped to the active org and wired to inventory domain tools (products, warehouses, stock movements). It replaces the stateless HTTP chat agent and adds the bottom-tab chat dock UI from the platform.

Reference implementation: sfab `apps/platform/src/agents/in-app/project/` (adapt project â†’ org context).

## Scope

- **Dependencies**: Add `@cloudflare/think`, `@cloudflare/shell`, `@cloudflare/codemode`, `@cloudflare/ai-chat`, `agents` (+ `agents/vite` plugin) to `web-tanstack/package.json`.
- **Vite config**: Add `agents()` plugin for `@callable()` decorator support in dev.
- **Wrangler config**: Add DO bindings for `OrgAgent` + `OrgChat`, migrations, `worker_loaders` (`LOADER`) for codemode. **No `WORKSPACE_R2`** â€” workspace uses SQLite-only spill (no large-file R2). Keep existing `R2_BUCKET` for user uploads.
- **Agent DO classes** under `src/agents/in-app/org/`:
  - `OrgAgent` â€” parent DO (1 per organization): shared `@cloudflare/shell` Workspace (SQLite only), chat registry, `org_memory` table, lazy default-chat spawn.
  - `OrgChat` â€” Think facet: session, compaction, model resolution, `SharedWorkspace` proxy, codemode + ERP tools + display tools.
  - `SharedWorkspace`, `OrgMemoryProvider`, system prompt, context assembly, page context (inventory entities).
  - ERP tool registry (products, warehouses, movements) inside codemode; display tools (`display-product-list`, `display-warehouse-list`, `display-memory`) as top-level peers.
- **Server entry**: Export DO classes + `CodemodeRuntime`; `routeAgentRequest` with auth + active-org gate at `/agents/org-agent/<organizationId>`.
- **Client wiring**: Bottom-tab chat dock (ported from platform): `OrgChatDockMount`, `BottomChatDock`, Zustand tab store, `useAgent` + `useAgentChat` WebSocket.
- **AppLayout**: Add optional `footer` prop for dock mount on protected routes.
- **Remove old agent infra** once wired: `src/lib/ai/agents/`, `respond.ts`, HTTP chat path as primary.
