# ALW-218 Bring in the Think-based project agent (ProjectAgent + ChatAgent DO pattern from sfab)

## Context

sfab-starter currently runs its AI agent as a stateless `AgentConfig` + `streamText` flow (`src/lib/ai/respond.ts`). The sfab platform repo has a full Think-based durable agent built on `@cloudflare/think` + the `agents` SDK — Durable Object classes (`ProjectAgent` parent + `ChatAgent` facets), shared workspace, git tools, multi-chat sessions, project memory, compaction, and recovery. This task ports that architecture into sfab-starter as the new project agent, replacing the current stateless approach.

Reference implementation lives in the sfab repo at:
- `apps/platform/src/agents/in-app/project/` — ProjectAgent, ChatAgent, supporting modules
- `apps/platform/vite.config.ts` — `agents()` vite plugin
- `apps/platform/wrangler.jsonc` — DO bindings, migrations, R2
- `apps/platform/src/server.ts` — DO exports, `routeAgentRequest`, auth gate

## Scope

- **Dependencies**: Add `@cloudflare/think`, `@cloudflare/shell`, `agents` (plus `agents/vite` for the Babel decorator plugin) to `web-tanstack/package.json`.
- **Vite config**: Add `agents()` plugin (required for `@callable()` decorator support in dev). See sfab `vite.config.ts`.
- **Wrangler config**: Add durable_objects bindings for `ProjectAgent` + `ChatAgent`, migrations (new_sqlite_classes), R2 bucket binding for workspace storage, and `worker_loaders` binding for codemode.
- **Agent DO classes**: Port a simplified version of sfab's `ProjectAgent` + `ChatAgent` into sfab-starter. This includes:
  - `ProjectAgent` — parent DO, owns shared workspace (`@cloudflare/shell` Workspace), chat registry, project memory table, lazy default-chat spawn.
  - `ChatAgent` — Think-based child facet. Owns per-chat session, compaction, model resolution, workspace proxy via `SharedWorkspace`, git tools, codemode tool wrapper.
  - `SharedWorkspace` — proxy forwarding child workspace calls to parent.
  - `ProjectMemoryProvider` — WritableContextProvider backed by parent storage.
  - System prompt builder, context assembly, page context.
  - Tool registry (codemode + git tools + display tools), adapted for starter's simpler domain.
- **Server entry** (`src/server.ts`): Export DO classes for wrangler resolution. Add `routeAgentRequest` dispatch with auth gate (session check via `@workspace/auth`). Remove or redirect the old `/api/chat` route if it conflicts.
- **Client wiring**: Switch the chat UI from the current `agentRespond` / `streamText` HTTP flow to a WebSocket-backed Think client connection. The client should connect to `/agents/project-agent/<projectId>` via the agents SDK's `useAgent` or equivalent hook.
- **Remove old agent infrastructure**: Delete or depreate `src/lib/ai/agents/`, `src/lib/ai/respond.ts`, `src/lib/ai/tools/`, and the skill-service system once the new agent is wired. Keep `src/lib/ai/utils/` (compaction, stream helpers) only if still referenced.

## Acceptance criteria

- [ ] AC-1: `@cloudflare/think`, `@cloudflare/shell`, `agents` are pinned in `apps/web-tanstack/package.json` and `pnpm install` succeeds. verify: `grep -E '@cloudflare/(think|shell)|"agents"' apps/web-tanstack/package.json`
- [ ] AC-2: `vite.config.ts` includes the `agents()` plugin and `vite dev` starts without decorator syntax errors. verify: `pnpm dev` boots cleanly.
- [ ] AC-3: `wrangler.jsonc` declares durable_objects bindings for ProjectAgent and ChatAgent, matching migrations, an R2 bucket for workspace, and `worker_loaders`. verify: `grep -n 'ProjectAgent\|ChatAgent\|worker_loaders\|WORKSPACE_R2' wrangler.jsonc`
- [ ] AC-4: `src/server.ts` exports ProjectAgent and ChatAgent classes, dispatches `/agents/` requests via `routeAgentRequest` with an auth gate, and `pnpm dev` serves the app. verify: opening the app and navigating to a chat connects via WebSocket without errors.
- [ ] AC-5: Agent DO classes exist under `src/agents/` (or equivalent) — ProjectAgent with shared workspace + chat registry + project memory, ChatAgent extending Think with session/compaction/tools/model resolution. verify: `ls src/agents/` shows the module structure; TypeScript compiles without errors.
- [ ] AC-6: Chat UI connects to the agent via WebSocket (agents SDK client), sends a message, and receives a streamed response. The old `agentRespond` / `streamText` HTTP path is no longer the primary chat route. verify: open chat, send a message, see streamed response in the UI.
- [ ] AC-7: Shared workspace works — agent can read/write files via the workspace proxy and they persist across chat sessions under the same project. verify: agent writes a file in one chat, reads it back in another.
- [ ] AC-8: Project memory works — `set_context` / `display-memory` round-trip through the parent DO so all chats in a project see the same memory. verify: agent saves memory in chat A, reads it in chat B.
- [ ] AC-9: Old agent infrastructure (`src/lib/ai/agents/`, `respond.ts`, `tools/`, skill service) is removed or clearly deprecated with no dead imports. verify: `grep -r 'agentRespond\|generalAgent\|getAiTools' src/` returns nothing (or only commented-out references).

## Non-goals

- Porting the full sfab tool surface (contractors, review, PRs, tasks, docs, logs, repos, activity). The starter gets a minimal codemode tool set — adapt or stub as needed.
- Porting ReviewAgent, WalkthroughRun, ContractorRun, or any sandbox/contractor infrastructure.
- Multi-org or multi-project tenancy beyond what the starter already supports.
- Changing the starter's auth system, DB schema, or UI component library.
- Matching sfab's full compaction/recovery/chatRecovery config exactly — start simple, tune later.

## Boundaries

- ✅ Always: base the agent code on the sfab reference implementation — copy the pattern, simplify for starter's needs.
- ✅ Always: keep the starter's existing auth (`@workspace/auth`) — wire the Think agent's `onConnect` to the same session resolution.
- ✅ Always: keep pnpm as the package manager; match existing formatting (biome.jsonc).
- ⚠️ Ask first: if the port requires changes to shared workspace packages (`@workspace/types`, `@workspace/core`).
- ⚠️ Ask first: if removing the old agent infra would break other features that haven't been ported yet.
- 🚫 Never: copy sfab-specific business logic (contractors, review, walkthrough, MCP server).
- 🚫 Never: change the sfab repo — all work happens in sfab-starter.
