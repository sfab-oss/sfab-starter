# ALW-299 Generalize the inventory vertical into the neutral base + expose pack seams (ADR-0010 §E)

## Context

The template still ships an inventory-vertical worked example (products, warehouses, stock movements, inventory AI tools, `/inventory` routes). [[ALW-47]] originally scoped this as "strip the vertical"; **ADR-0010 (Amendment 2026-06-16 §A)** re-scoped it to **capability-complete + domain-generic** — generalize, don't delete. **Amendment 2026-06-18 §E** gives the concrete target: a **country-neutral base** shaped around the Tier-1 parts list, with country/vertical specialties living in **packs**.

**The seams + hub schema are DESIGNED and MERGED in [[ALW-328]]** — starter **ADR-006** (`docs/decisions/006-transaction-core.md`) + **`docs/architecture/transaction-core.md`**. This task **implements that design**; it does not re-decide it.

**Scope revision 2026-07-01 (planning review):** the original single task was split into three sequential deliverables so each PR is reviewable and the user-visible template transformation lands first:
1. **This task** — money foundation + the `documents`/`line_items` hub + `sequences` + folio-atomic finalize + event dispatch + de-verticalization + the neutral demo.
2. [[ALW-354]] — the settlement engine (`payments`, `payment_allocations`, `recordPayment`, projections, conservation).
3. [[ALW-355]] — the `customer_credit` wallet (saldo a favor / anticipos / walk-in matching).

The relevant red-team invariants for THIS slice are **C5** (fiscal freeze semantics) and **C9** (structural families). C1/C2/C4/C6 → [[ALW-354]]; C3 → [[ALW-355]]. (Note: ALW-328's outputs define C1–C7 and C9 only; there is **no C8** — earlier "C1–C9" phrasing was a numbering slip, not a missing requirement.) The AI-review forward notes are routed: (a) allocation upsert → ALW-354; (b) walk-in matching → ALW-355; (c) `sale_completed` firing → ALW-354.

**Engineering constraints (binding, from the 2026-07-01 planning review):**
- **D1 has no interactive transactions.** Production D1 rejects `BEGIN/COMMIT`; `db.transaction()` passes local miniflare but fails on real D1. Every multi-write mutation is one **`db.batch([...])`** of pre-assembled statements (reads before batch assembly; the full-scan rebuild model is the correctness backstop). `db.transaction()` is banned in review.
- **Test harness:** D1-touching tests run through the existing `apps/web` workers-pool harness (`@cloudflare/vitest-pool-workers` + `readD1Migrations("packages/db/drizzle")`). `packages/core` has no test runner and must NOT gain one here (that's [[ALW-305]]); pure `core/money` unit tests live as a no-DB file under `apps/web/test/`. Verify = full `pnpm typecheck && pnpm lint:check && pnpm test`.
- **Migration safety:** [[ALW-111]] (the `PRAGMA foreign_keys=OFF` guard) lands **before** this task — Phase-0 is a money table-rebuild and the de-verticalization drops FK-bearing tables, exactly the ADR-0008 failure class. Follow `defer_foreign_keys` conventions.

New projects should `pnpm install && dev` and pass `typecheck`/`lint`/`test` with **nothing to delete** before building their own domain.

## Scope

### Keep (capability-complete)
- Auth + org/multi-tenancy; D1 + Drizzle scaffold; search, uploads, email; the in-app AI assistant with example tools; shadcn UI + sidebar + dashboard shell.

### Money foundation (reverses [[ALW-327]] D3, per ADR-006)
- Branded **`moneyMinor`** integer customType in `db/utils.ts` (minor units); **remove** the float `numeric → number` `money` type entirely.
- `packages/core/src/money.ts`: add/sub/allocate/round, basis-point rates/discounts, round-per-line with header = exact Σ lines, ISO-4217 minor-exponent map. **No `@workspace/db` import** (stays runtime-pure).
- Migrate Catalog `price`/`cost` with the **full consumer cascade**: `contract` Zod (integer minor units), `core` selects, UI call sites (note: `apps/web/src/routes/_protected/inventory/index.tsx` already calls `majorToMinor(...)` assuming float storage — remove the wrapper or values double-scale; reuse the existing `packages/ui/src/lib/money.ts`), seeds/tests/e2e fixtures. Migration does no value transform on table-rebuild — add a `round(price*100)` backfill or document the fresh-DB-only waiver.
- Update the money-convention line in `AGENTS.md` + `.claude/CLAUDE.md` (keep byte-identical).

### The hub (per `transaction-core.md` §1–§3)
- One `documents` + one `line_items` table: `type`/`family`/`direction`/`status`, nullable `entityId` + counterparty snapshot, MoneyMinor columns, **projection columns included now** (`amountPaid`/`balanceDue`/`paymentStatus` at defaults — ALW-354 fills them; avoids another table-rebuild), numbering, temporal, lineage, `metadata`; line items with nullable `productId` + snapshot, qty/unitPrice/discount, generic tax (`taxRate` bps / `taxCode` / `taxMode` / `taxAmount` / `taxableBase`), `fulfillmentMode`, **bare nullable `warehouseId` (no FK, no location table — that's a pack)**, `metadata`.
- **`family` = persisted column + DB CHECK linked to `type`** (C9); `core.documentFamily()` with **no default branch** (unknown type throws). SQLite can't ALTER a CHECK — define the **complete type↔family map up front** (including `credit_note` and other types whose flows land later) and test that the SQL CHECK and the TS resolver agree.
- Transition guards (draft→finalized→voided; void = reversal marker; consume the reserved `document:void` RBAC key). Immutable fiscal successors.
- `sequences(orgId, key)` + **folio-atomic `finalize`**: freeze fiscal lines/totals/tax/identity (**C5** — payment/posting columns may advance later), snapshot the frozen tax context into `metadata.tax`, compute `taxTotal` **excluding withholdings**, and bump the folio — all in ONE `db.batch()`. Structure the batch as an explicit statement array with a **seam-marker comment** where a pack later appends critical statements (no `registerCriticalHandler` API — the extension mechanism is designed with its first consumer in [[ALW-350]]).
- **Event dispatch:** `core.emitDomainEvent` writes `kind:event` rows inside the batch + **afterCommit** fan-out to real base consumers (activity timeline via `schema/activity.ts`, event log). `document_finalized` fires here; `sale_completed` is defined in [[ALW-354]]. The generic afterCommit dispatch IS the posting seam — a GL pack subscribes later (C7); no GL hook is built here.
- `shouldAffectStock(line)` = catalog `tracksInventory` × line `fulfillmentMode` — the **gate ships; the stock handler is [[ALW-350]]'s** (pack-owned).
- Remaining §9 seams with a base consumer or zero cost: namespaced `metadata`, header/line `discount`, nullable `entityId` + `creditLimit` column, reversal markers, `assertPostingAllowed`, financial-family split seam, functional-currency optional columns.

### Generalize the vertical (domain-generic, parts-list-aligned)
- **Remove** `movements` + `stock_levels` + `performStockMovement` with the full cascade: `core/catalog/products.ts` (`totalStock` join, `getProductMovements`, `getDashboardMetrics`), `contract` movement schemas + `totalStock`, the `/movements` Hono route, `products.test.ts`, and the `packages/registry` `today-overview` inventory-feed mocks.
- Neutralize naming: `/inventory` routes + Hono mount + hardcoded paths in `core/catalog/search.ts` + agent persona (`packages/agent/src/context/system-prompt.ts`) + `providers/page-context.tsx` / `chat/page-context-chip.tsx` + sidebar/search UI → land on **Catalog + Documents**. Regenerate `routeTree.gen.ts`.
- One neutral end-to-end demo: catalog CRUD + quote → order → invoice → **finalize (folio drawn, totals frozen)** + activity feed + an AI tool call. Payment recording arrives with [[ALW-354]]. Money/document mutations stay off the agent (guard.ts convention).
- README: document the **base contract** (what ships vs what the AI adds); remove "delete example features to start."

## Non-goals

- The settlement engine (`payments`, `payment_allocations`, `recordPayment`, conservation, credit-note dispositions) — [[ALW-354]].
- The `customer_credit` wallet, walk-in matching, anticipos — [[ALW-355]].
- Inventory `movements` + the stock critical handler + the pack critical-handler **extension mechanism** — [[ALW-350]] (the first consumer shapes the API; this task leaves only the seam-marker).
- A base GL posting hook, `line_tax_components` (multi-tax), CFDI sidecar, financial-family table split, functional-currency implementation, gapless numbering — deferred per ADR-006.
- Removing the worked example entirely (forbidden); designing seam shapes (consume ALW-328's); the registry/pack mechanism ([[ALW-324]], done); a package-local core/db test harness ([[ALW-305]]).

## Boundaries

- ✅ Always: keep auth/search/uploads/AI infrastructure; generalize flavor, don't strip capabilities; domain-neutral naming; seams additive and sourced from ALW-328; `db.batch()` atomicity; `defer_foreign_keys` migrations.
- ⚠️ Ask first: changes to shared packages beyond the scoped cascade; dropping a cross-cutting capability; any deviation from `transaction-core.md`'s shapes.
- 🚫 Never: `db.transaction()`; reintroduce recipe/`add` layers; delete the worked example; change the sfab platform repo; ship payments/wallet tables here (they're ALW-354/355); a base GL or multi-tax; hand-edit generated files (`routeTree.gen.ts`, registry `generated.ts` — regenerate).
