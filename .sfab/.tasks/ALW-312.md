# ALW-312 Transform sfab-starter into the AI-native ERP golden base (manual, comment-driven)

## Context

The single umbrella task for the **manual** transformation of `sfab-starter` into the golden, AI-native ERP base the fabricate pipeline ([[ALW-306]]) instantiates. This is product/UX work done **by hand, step by step, human-in-the-loop** — progress is captured as **comments on this task**. Distinct from the platform machinery in epic [[ALW-306]]; lives in the [[ALW-323]] Group alongside the scoped deliverables ([[ALW-299]], [[ALW-354]], [[ALW-355]], [[ALW-304]], [[ALW-305]], [[ALW-332]], [[ALW-350]]), which stay independent.

**Rescoped 2026-07-01 (planning review)** — much of the original scope has since been delivered or superseded by other work:
- **ERP scope-plan** → SUPERSEDED. ADR-0010 Amendment §E (the Tier-1 parts list) + ADR-006/`transaction-core.md` (the merged Transaction Core design) now *are* the ERP scope definition; no separate scope doc is needed.
- **Component-registry cleanup / separate package / fabricate-omitted** → DELIVERED by [[ALW-324]] (`packages/registry` producer + build step) and [[ALW-351]] (gallery); `packages/registry` + `apps/docs` are already in `.sfab/fabrication.json`'s `drop` list. What REMAINS from that bullet: the **hard-part components** themselves.
- Remaining live scope: hard-part components, live-docs system, AI Gateway, golden-codebase cleanup (below).

## Scope (worked manually; each a comment thread, possibly spun out later)

- **Hard-part components** in the registry with clean APIs — don't re-wrap what shadcn composes well; add the genuinely hard ones: markdown editor, PDF viewer, Excel/spreadsheet viewer, etc. AI gets freedom on basics (layout), pre-built composables for the hard parts.
- **Live documentation system:** format/templates, how it's maintained, where it lives, versioning (ties to ADR-0010 Decision 5 / capability docs).
- **Cloudflare AI Gateway** (model observability/caching) — verified 2026-07-01: NOT yet integrated; the only "gateway" in the repo is the Vercel AI SDK's `gateway()` model resolver in `packages/agent/src/inference/chat-models.ts`, and no `ai_gateway` binding exists in the wrangler configs. Add value, not resources-for-their-own-sake.
- **Golden-codebase cleanup:** de-"vibe-code" existing parts to the quality bar.
