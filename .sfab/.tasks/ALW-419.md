# ALW-419 Dogfood UX polish: shell chrome, settings layout, dialog padding, mobile nav/chat

## Context

Desktop + mobile dogfood QA on the starter (`active/starter-dogfood-qa/ISSUES.md`) confirmed several UX defects that make the app feel unfinished: duplicated leaf breadcrumbs, oversized header CTAs, flush dialog padding, mobile sidebar that stays open after nav, desktop-style chat chrome on phone, and Organization Settings that stretch full-width / lack spacing (and don’t match the platform or the starter’s own `settings-page` registry block).

Evidence: `artifacts/browser-qa/starter-20260708-fresh/` and `starter-20260708-mobile-qa/`.

## Scope

- Fix leaf breadcrumb + `ShellHeaderTitle` double-label on routes that show it (Today, Settings); keep invoice-style breadcrumbs (parent + distinct leaf) as the good pattern.
- Shrink page-header primary actions (**Add Product**, **New Invoice**, and the same pattern elsewhere) to a compact size on desktop and mobile — not full “hero” buttons in the shell header.
- Add inner padding to the Add Product (create-product) dialog so fields aren’t flush to the dialog edge.
- Close the mobile app sidebar automatically when a nav link is tapped (match platform `useCloseMobileSidebarOnNavigate`).
- Align mobile org-agent chat header with the registry `chat-dock` `MobileChatHeader` (chevron-down dismiss, centered title, sheet-native chrome — not desktop minimize/X row).
- Organization Settings layout pass:
  - Constrain content width on desktop (`max-w-4xl` already exists in places — ensure it applies and doesn’t feel full-bleed).
  - Add horizontal page padding + vertical gap between settings cards on mobile (and desktop if missing).
  - Adopt the platform / registry **settings sub-nav** pattern (`SettingsSectionLayout`: General / Members as sections; URL routes preferred like platform, or registry client sections as interim) so settings aren’t one endless scroll.

## Acceptance criteria

- [ ] AC-1: Leaf routes that previously showed `Today Today` / `Settings Settings` show the page name **once** in the shell header (breadcrumb-only like Catalog/Documents/platform, or title-only — not both with the same string). verify: Today + Settings desktop and mobile screenshots
- [ ] AC-2: Catalog **Add Product** and Documents **New Invoice** header actions use a compact control size (e.g. `size="sm"` or equivalent) and do not dominate the header row on iPhone width. verify: catalog + documents headers desktop + mobile
- [ ] AC-3: Add Product dialog has clear inner padding between form content and dialog edges on desktop and mobile. verify: create-product dialog open
- [ ] AC-4: On mobile, tapping a Platform nav item (Home / Catalog / Documents / Settings) navigates **and** dismisses the sidebar overlay without requiring Escape / outside tap. verify: open sidebar → tap Catalog → sidebar closed on catalog
- [ ] AC-5: On mobile, open org agent chat shows sheet-style header with chevron-down (or equivalent) to dismiss, matching `chat-dock` `MobileChatHeader` intent — not the desktop window-control cluster as the primary dismiss affordance. verify: mobile agent open screenshot vs registry block
- [ ] AC-6: Organization Settings on desktop has a readable max content width (not stretched edge-to-edge of the main column) and on mobile has outer page padding plus visible gap between setting cards. verify: settings desktop + mobile
- [ ] AC-7: Organization Settings uses a sectioned layout with sub-nav (at least General vs Members), following platform / `packages/registry/.../settings-page` — not a single undifferentiated scroll of every card. verify: settings with sub-nav visible desktop; mobile section switcher works

## Non-goals

- Full catalog **data table** redesign (filters, removing Columns, resource-table migration) — track separately; user will spec ideal table.
- Full **invoice create/edit** redesign (including a polished line-item composer) — track separately; function works, layout needs its own pass.
- Chat **approvals / codemode** wiring QA (tool-level `needsApproval` vs codemode approvals) — dedicated follow-up session.
- Price “minor units” labeling, chat history titles, approval UUID display, agent trace noise — parked unless they fall out of the above ACs.

## Boundaries

- ✅ Always: reuse existing shell / registry / platform patterns (`ShellHeader`, `SettingsSectionLayout`, `MobileChatHeader`, platform sidebar close-on-nav) instead of inventing new chrome.
- ✅ Always: verify on both desktop and iPhone-width (or `set device "iPhone 16"`) before handoff.
- ⚠️ Ask first: changing settings URLs / introducing `/settings/general` + `/settings/members` if it breaks deep links or demos.
- 🚫 Never: redesign catalog table or invoice document composer in this task.
- 🚫 Never: hand-edit `.sfab/.tasks/*`.
