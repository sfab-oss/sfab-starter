# ALW-527 Replace deprecated createServerFn().inputValidator() with .validator()

## Context

Docs app build emits a TanStack Start deprecation warning from the `tanstack-start-core::server-fn` plugin:

```
createServerFn().inputValidator() is deprecated. Use createServerFn().validator() instead.
```

Source: `apps/docs/src/routes/docs/$.tsx` (the docs `serverLoader`). Build still succeeds; this is noise that will become a hard break when the old API is removed.

## Scope

- Rename `.inputValidator(...)` → `.validator(...)` on the docs `serverLoader` in `apps/docs/src/routes/docs/$.tsx`.
- Confirm no other `inputValidator` call sites remain in the starter repo.
- Verify `pnpm --filter docs build` no longer prints the deprecation warning.

## Acceptance criteria

- [ ] AC-1: `apps/docs/src/routes/docs/$.tsx` uses `.validator(...)` (not `.inputValidator(...)`) on the docs `createServerFn` serverLoader. verify: `rg inputValidator apps/` is empty.
- [ ] AC-2: `pnpm --filter docs build` completes without the `inputValidator() is deprecated` warning. verify: build log has no `inputValidator` / deprecation lines from `tanstack-start-core::server-fn`.

## Non-goals

- Same fix in `sfab-oss/sfab` (file a follow-up after this lands).
- Broader TanStack Start upgrades or API migrations beyond this rename.
- Changes to `apps/web` `createServerFn` usages (they do not use `inputValidator`).

## Boundaries

- ✅ Always: one-line API rename; keep validator body and handler unchanged.
- ⚠️ Ask first: any change beyond the rename / verification.
- 🚫 Never: edit `.sfab/` mirrors by hand; touch the platform repo in this task.
