# ALW-432 Surface the transaction core in the UI — entities, wallet, document types, payments (Group)

## Context

The backend transaction core ([[ALW-354]], [[ALW-355]]) is well ahead of the UI. The 2026-07-08 surface sweep found whole subsystems with full org-scoped APIs and **zero or minimal UI**:

- **Entities** (customers/suppliers): list/get/create endpoints, no screen anywhere; documents show free-text `entityName` / "Walk-in" only.
- **Customer-credit wallet**: balance, ledger entries, deposit / redeem / redeem-by-reference — API-only.
- **Document types**: contract supports 10 types × sales/purchase directions; the UI creates only sales invoices.
- **Payments**: standalone list/get/create/reverse endpoints; only reachable via one form on a document detail, no payments view, no reversal UI.

For the golden ERP base this is the difference between "an invoice demo" and "an ERP". This Group organizes closing that gap.

## Scope

Children:
- Entities directory & picker (foundation — wallet and documents build on it)
- Customer-credit wallet UI
- Document types & directions beyond the sales invoice
- Invoice create/edit UX redesign ([[ALW-430]], pre-existing)

## Non-goals

- New backend capabilities beyond small API completions the UI needs (e.g. entity update).
- POS pack ([[ALW-350]]) — it consumes these surfaces, it doesn't build them.
