# ALW-587 i18n: close remaining hardcoded English app chrome (post Spanish agent bootstrap)

## Context

During a Spanish non-technical owner roleplay against a fresh clone of sfab-starter (`active/agua-pura-agent-test/`), the bootstrap agent and the live UI still showed hardcoded English in places that ALW-457 catalogs already cover (or should). Fix after the roleplay agent finishes - do not interrupt that run.

Notable misses / smells:
- Login chrome initially EN (Login / Email / Password) until the agent patched it ad hoc.
- `record-payment-dialog.tsx` still had hardcoded "Record payment", "Saving…", "Pay" in the roleplay tree while main already has `m.documents_record_payment()` etc.
- Agent invented `apps/web/src/lib/copy-es.ts` instead of using Paraglide `m.*` + locale cookie - parallel string surface that will drift from `packages/i18n`.
- Registry blocks remain EN-only (tracked separately as [[ALW-571]]).

## Scope

- Audit starter app surfaces for remaining JSX/string literals that should be `m.*` (auth, documents payment dialog, empty/error chrome, buttons).
- Prefer wiring existing EN/ES catalog keys; add keys only where missing.
- Document that agents should localize via Paraglide / language switcher, not a one-off `copy-es.ts`.
- Optionally add a short note to `docs/guides/i18n.md` / i18n skill: do not invent parallel copy bags.

## Acceptance criteria

- [ ] AC-1: Document/payment dialog chrome uses catalog keys (`documents_record_payment`, pay/cancel/saving) with no "Record payment" literals in `apps/web` (verify: `rg 'Record payment' apps/web`)
- [ ] AC-2: Auth login labels under ES locale match catalog (`auth_login` / `auth_email` / `auth_password`) without a bespoke `copy-es` bag (verify: browser ES + ripgrep)
- [ ] AC-3: No new parallel Spanish copy module in `apps/web/src/lib/` that duplicates Paraglide (verify: code review)
- [ ] AC-4: Short i18n skill/guide note: agents must use `packages/i18n` + `m.*`, not ad-hoc ES string maps

## Non-goals

- Registry block demo copy ([[ALW-571]]).
- Rewriting the agua-pura roleplay project itself (throwaway test tree).
- New locales beyond EN/ES.

## Boundaries

- Always: catalog-first; reuse existing keys; keep `packages/ui` locale-agnostic (label props only).
- Ask first: expanding scope into every toast/error string in one PR.
- Never: put Paraglide into `packages/ui`; weaken locale cookie strategy.
