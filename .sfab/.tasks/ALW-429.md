# ALW-429 Command-K palette: wire real global search (pages + catalog + documents)

## Context

The ‚åòK palette (`components/search/search-command.tsx`) looks finished but is still a stub for entity search. [[ALW-423]] QA (2026-07-08) confirmed the **static page list is already complete** (Home, Catalog, Entities, Documents, Settings) ‚Äî earlier notes that Documents was missing are stale. Remaining gaps: no product/document entity search; dead scaffolding (`isLoading` hardcoded `false`, a 300ms debounce that only feeds a local string filter); the query string **persists** when the dialog closes and reopens (finding S-2); and the **search input chrome looks wrong** (finding S-7) ‚Äî a short pill nested inside an extra `border-b` flex wrapper, so it doesn‚Äôt span the dialog width.

For an AI-native ERP base the palette should be the fast path to any record.

## Scope

- Keep the static page list derived from the nav definition (already includes Documents; avoid duplicating a second hardcoded list).
- Wire live entity search: products via the existing catalog search endpoint; documents (folio/entity name) via a search-capable endpoint (add one if the documents API lacks it).
- Grouped results (Pages / Products / Documents) with icons, loading state, empty state, and Enter-to-navigate to the right detail route.
- Reset `query` / `debouncedQuery` when the dialog closes (`onOpenChange(false)`).
- Either wire `isLoading` to real async work or remove the dead scaffold.
- Fix search-input layout: drop the nested pill / extra header wrapper so `CommandInput` spans the dialog like a normal cmdk header (full-width, no double chrome).
- Keep keyboard behavior (‚åòK, Ctrl-K, `/`, input-focus guard) intact; keep response snappy (debounce + request cancellation).

## Acceptance criteria

- [ ] AC-1: Typing a product name surfaces catalog results that navigate to `/catalog/$id`.
- [ ] AC-2: Typing a document folio or entity name surfaces document results that navigate to `/documents/$id`.
- [ ] AC-3: All app pages, including Documents, appear as page results.
- [ ] AC-4: Loading, empty, and no-results states render correctly; the `isLoading` scaffold is either real or removed.
- [ ] AC-5: Keyboard-only use works end-to-end (open, navigate results, select, close).
- [ ] AC-6: Closing and reopening the palette shows an empty query (no sticky previous search). verify: type a query, Esc, ‚åòK again.
- [ ] AC-7: Search input spans the dialog header cleanly (no short nested pill / double border chrome). verify: screenshot of open palette.

## Non-goals

- Command/action execution from the palette (create product, etc.) ‚Äî possible follow-up.
- Full-text search infrastructure; simple prefix/LIKE search is fine for the base.
