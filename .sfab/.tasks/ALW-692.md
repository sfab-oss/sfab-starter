# ALW-692 Bump sfab-starter knip 5 → 6 (TS 7 peer range)

## Context

sfab is on TypeScript 7.0.2 ([[ALW-573]]) and needed knip 6 because knip 5 peers `typescript: '>=5.0.4 <7'` and crashes on TS 7. sfab-starter is already on TypeScript 7.0.2 ([[ALW-473]]) but still pins knip `^5.88.1`, so `pnpm check:dead-code` is broken or fragile the same way.

## Scope

- Bump `knip` to **6.27.0** (or `^6.27.0`) — clears the 2-day `minimumReleaseAge` gate; do **not** take 6.28/6.29 until mature.
- Fix knip 6 config fallout (hints / unlisted binaries / unused-export false positives) so `pnpm check:dead-code` exits 0.
- Mirror the mechanical pattern from sfab PR #619 where applicable.

## Acceptance criteria

- [ ] AC-1: lockfile resolves knip 6.x (≥6.27.0, no exclude needed for release-age).
- [ ] AC-2: `pnpm check:dead-code` passes on the branch.
- [ ] AC-3: `pnpm typecheck`, `pnpm lint:check`, `pnpm test`, `pnpm build`, `pnpm check:cycles` still pass.
- [ ] AC-4: PR notes any starter-specific knip 6 fallout vs sfab.

## Non-goals

- Broader dependency bumps unrelated to knip.
- Changing whether knip runs in pre-commit (starter pre-commit currently skips it).

## Boundaries

- ✅ Always: keep the change small; prefer config/ignore fixes over suppressing real unused code without cause.
- ⚠️ Ask first: if knip 6 forces unrelated major bumps.
- 🚫 Never: add a `minimumReleaseAgeExclude` for a knip version that is already mature (use 6.27.0).
