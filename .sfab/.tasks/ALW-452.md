# ALW-452 Fix page-context chip never mounting on first paint

## Launch cut (2026-07-10)

**Status: soft stretch** under [[ALW-479]] (also filed from [[ALW-422]] QA). In scope for launch polish; not a hard gate.

## Context

ALW-427 found the chat page-context chip missing on Catalog list and product detail (`Artisan Widget`) even though routes call `useSetPageContext`. In `apps/web/src/components/providers/page-context.tsx`, `useSetPageContext` only calls `setPageConfig` when the config JSON *changes*; the first mount is skipped, so `pageConfig` stays `undefined` and `PageContextChip` never renders.

## Scope

- Set page context on first mount (and keep cleanup-on-unmount behaviour).
- Verify chip shows title (e.g. product name) with pin/dismiss on catalog detail and at least one other route (documents/entities).

## Acceptance criteria

- [ ] AC-1: Opening `/catalog/:id` shows a page-context chip with the product name above the chat input (verify: browser).
- [ ] AC-2: Pin and dismiss controls work; dismiss shows "Add context"; restore brings the chip back.
- [ ] AC-3: Navigating away clears context; navigating to another contextual route updates the chip.

## Non-goals

- Broader chat UX (ALW-401) beyond this chip wiring bug.

## Boundaries

- ✅ Always: minimal fix in page-context provider + browser evidence.
- 🚫 Never: change what routes publish as context without need.
