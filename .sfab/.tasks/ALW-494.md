# ALW-494 Own Biome ruleset — drop Ultracite, bump Biome, smell lint gates

## Context

Both `sfab-starter` and `sfab` currently `extends` Ultracite's Biome presets (`core` / `react` / `next`) and pin `@biomejs/biome` at 2.3.x. Starter D8 (ALW-327) already decided to drop Ultracite and self-maintain Biome; ALW-326 deferred the concrete ruleset after a brief remove-then-revert. We also want lint gates that catch documented code smells (starting with the `useEffect`-sync smell) as prevention alongside AI review.

## Scope (Group)

Coordinated work across both repos, **starter first**, then platform:
1. Fork Ultracite's useful rules into a repo-owned `biome.jsonc` (no `ultracite` dependency).
2. Bump `@biomejs/biome` (target current latest at implementation time; today 2.5.x).
3. Trim / lighten rules we don't want; drop Next-oriented preset unless justified.
4. Add smell-oriented Biome rules where enforceable (e.g. restrict `useEffect` imports with justified `biome-ignore`).
5. Port the same baseline to `sfab`, preserving platform-only overrides (`lib/server` import guard, unused-import fix:none, etc.).

## Shape

- Deliverable 1: starter (this Group's first child).
- Deliverable 2: platform (`sfab`) after starter lands.

## Non-goals (Group-level)

- Replacing AI review — lint is additive prevention.
- Encoding database-smells / migration judgment in Biome (not AST-enforceable).
- A shared published `@sfab/biome-config` package (unless we later decide we need one).
