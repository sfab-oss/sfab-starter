# ALW-494 Own Biome ruleset — drop Ultracite, bump Biome, smell lint gates

## Context

Both `sfab-starter` and `sfab` currently `extends` Ultracite's Biome presets (`core` / `react` / `next`) and pin `@biomejs/biome` at 2.3.x. Starter D8 (ALW-327) already decided to drop Ultracite and self-maintain Biome; ALW-326 deferred the concrete ruleset after a brief remove-then-revert.

## Scope (Group)

Coordinated work across both repos, **starter first**, then platform:
1. Fork Ultracite's useful rules into a repo-owned `biome.jsonc` (no `ultracite` dependency).
2. Bump `@biomejs/biome` (target current latest at implementation time; today 2.5.x).
3. Trim / lighten rules we don't want; drop Next-oriented preset.
4. **Later:** smell-oriented Biome rules (`useEffect` justify-gate, etc.) after the Ultracite exit is green — requires a baseline pass in app code.
5. Port the same baseline to `sfab`, preserving platform-only overrides.

## Shape / order (settled)

1. [[ALW-495]] Starter: drop Ultracite, own config, bump Biome (no smell gates).
2. Platform deliverable (TBD): same for `sfab`.
3. Smell-lint deliverable(s) (TBD): last — justify existing effects / encode enforceable smells.

## Non-goals (Group-level)

- Replacing AI review — lint is additive prevention.
- Encoding database-smells / migration judgment in Biome (not AST-enforceable).
- A shared published `@sfab/biome-config` package (unless we later decide we need one).
