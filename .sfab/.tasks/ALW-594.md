# ALW-594 Template-init interview v2 — intake contract: answers → product brief → reshape task slate

## Context
`docs/template-init.md` ([[ALW-461]]) defines the adoption interview — 8 product-owner questions plus an answers→codebase-actions translation table. [[ALW-568]] (merged, PR #84, after the live agent-init audit) already upgraded it: interview Q&A must be persisted under `docs/`, a separate whole-app transform plan file is required, and execution is eager. What remains undefined is the **platform-side contract**: the chat-first intake ([[ALW-591]]) runs the interview with the *user* before/at fabrication, so both sides must share one artifact — what the interview produces, where it persists, and how it seeds the initial reshape tasks mechanically.

## Scope
- Consolidate the `docs/` Q&A + transform-plan artifacts from [[ALW-568]] into a defined "product brief" contract (shape, path, lifecycle) that the platform intake can write at fabrication and the in-repo reshape agent reads on first run.
- Adapt the question set for a conversational pre-fabrication intake — ordering, one-question-at-a-time flow, smallest-win emphasis, stop conditions (enough to start ≠ complete spec) — without forking the guidance.
- Map brief → initial reshape task slate using the translation table, concretely enough that tasks can be generated mechanically; include one worked example.
- Decide (with [[ALW-591]]) whether `template-init.md` stays the single source the platform consumes, or the platform owns a copy — record the decision either way.

## Acceptance criteria
- [ ] AC-1: The interview guidance is usable both by an adopting agent and a conversational pre-fabrication intake (verify: doc sections address both consumers).
- [ ] AC-2: Product-brief artifact defined (shape, location, lifecycle), unifying the ALW-568 Q&A/plan files, and referenced from the doc.
- [ ] AC-3: Brief→task-slate mapping documented with at least one worked example.
- [ ] AC-4: Source-of-truth decision recorded and cross-referenced with [[ALW-591]].

## Non-goals
- Building any platform surface (that's [[ALW-591]]'s subtasks).
- Re-doing what [[ALW-568]] shipped (persistence, eagerness, whole-app reshape rules).
- Capability-pack mechanics (ADR-0017 / [[ALW-350]]).

## Boundaries
- ✅ Always: keep the doc honest about what exists in the repo today.
- ⚠️ Ask first: restructuring docs/ layout.
- 🚫 Never: reference features or packs the template does not ship.
