# ALW-461 Template release docs: README + agent init guidance

## Context

When we release `sfab-starter` as a public template, consumers will copy it and (often with an agent) reshape it into their product. The template is **built for the factory first**, but it will also ship and run **standalone** outside the factory.

Today the README may already cover how to run the app locally. What's missing for template release is **guidance for that copy → transform loop**: what this codebase is, how to take it somewhere else, and what an agent should ask the user before changing things. We want that guidance to be helpful without being overly prescriptive.

## Scope

- Review / refresh the README for template release: how to use and run the template (standalone), and a clear pointer that the template targets the factory but can be used separately.
- Add a **separate agent-oriented markdown file** (suggested init / transformation guide) that is not the README itself — e.g. something like `AGENTS.init.md` or `docs/template-init.md` (final name TBD).
- From the README, link to that agent guidance file.
- In the agent guide: a light suggested questionnaire / interview path (product intent, domains to keep vs strip, branding, auth/deploy targets, factory vs standalone, etc.), orientation to the codebase shape, and a recommended path for planning then transforming — guidance, not a rigid script.

## Acceptance criteria

- [ ] AC-1: README includes copy/use/run instructions suitable for template consumers (standalone), and states factory-first + standalone-usable positioning — verify: open README on a fresh clone
- [ ] AC-2: A dedicated agent-init / transformation guidance markdown file exists in the repo and is linked from the README — verify: follow the README link to the file
- [ ] AC-3: The agent guide covers (at minimum) codebase orientation, suggested early questions for the user, and a non-prescriptive path from answer → plan → transform — verify: read the guide
- [ ] AC-4: Tone stays guidance-level (not a mandatory checklist or rigid wizard) — verify: no "you must" workflow that precludes other approaches

## Non-goals

- Implementing the transformation itself, skills, or factory wiring.
- Building an interactive in-app questionnaire UI.
- Full product documentation beyond template bootstrap / init guidance.
- Deciding final public packaging / GitHub template settings (separate from this docs work unless already settled).

## Boundaries

- ✅ Always: keep factory-first positioning honest; prefer a separate agent guide over stuffing the README; leave room for humans and agents to diverge from the suggested path.
- ⚠️ Ask first: exact filename/location of the agent guide; how much existing README runbook to rewrite vs leave.
- 🚫 Never: make the guide a required one-true transformation ritual; encode factory-only assumptions that break standalone use.
