# ALW-427 QA: org-agent chat & approvals wiring — dock, tools, memory, files, mobile

## Context

The chat is the largest and most differentiating feature: multi-tab bottom dock, popup/fullscreen windows, history, org memory, page-context chips, attachments, voice, tool-result cards, sub-agent delegation, and a workspace file tree. The first pass validated the read path and surfaced QA-1: are codemode-level approvals and per-tool `needsApproval` both wired correctly, and how do they interact? ([[ALW-348]] merged the durable-approval pattern.) Mobile chat issues M-3/M-5 were addressed in [[ALW-419]] and need regression checks. [[ALW-401]] (chat UI/UX improvements) is an open draft this session should sharpen with evidence.

## Scope

- Approvals (the QA-1 centerpiece): trigger a tool-level `needsApproval` action (e.g. delete product) and a codemode-level approval; Approve and Reject each; approval survives reload/reconnect (durable pattern); approval card content quality (raw ids — tracker B-4).
- Dock mechanics: tab cap + overflow menu, new chat, draft→real promotion, history open/delete, popup vs fullscreen, close/minimize/restore.
- Conversation: streaming, stop-streaming, regenerate, copy, error + retry states, reconnect after killing the connection.
- Context features: page-context chip auto-attach/pin/dismiss on each route; org-memory dialog round-trip (agent writes memory, dialog shows it); attachments (drag-drop + menu); voice input round-trip.
- Tool cards: product-list card links resolve; delegate-run streams a child transcript; fallback card for unregistered tools.
- Files panel: workspace tree lazy-load, sub-agent run view, mobile files sheet.
- Org switching: chats remount and never leak across orgs.
- Mobile: FAB entry, chat header (M-5 regression), overlay usability (M-3).

## Acceptance criteria

- [ ] AC-1: Approval matrix documented and verified with evidence: {tool-level, codemode} × {approve, reject, reload-mid-approval}.
- [ ] AC-2: All dock/window/history mechanics verified desktop + mobile with evidence.
- [ ] AC-3: Page-context chip, org memory, attachments, and voice each verified round-trip.
- [ ] AC-4: Tool cards + delegate-run + files panel verified with evidence.
- [ ] AC-5: Org-switch isolation of chats verified.
- [ ] AC-6: Findings logged; UX-grade findings routed into [[ALW-401]] instead of new tasks where they fit.
