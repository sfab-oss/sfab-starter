# ALW-350 Build the first capability pack (POS) + complete the pack-install mechanism (ADR-0017)

## Context

[[ALW-324]] landed the capability-pack registry *foundation* — the repo as a `registry.json` **producer**, the single shadcn `RegistryItem` primitive discriminated by `meta.sfabKind`, the docs gallery + Layer-1 harness, and the drift-checked build step (`packages/registry`). But it merged with **every item as `sfabKind: "block"`**: the `pack` value is a defined-but-dead seam, and the actual capability-pack mechanism (code-bearing packs, ephemeral install skills, provenance) was **deferred** rather than built.

This task completes that mechanism and proves it end-to-end with the flagship **POS** pack as the first real vertical — POS doubles as the proving pack ALW-324's ACs called for. The `meta.sfabKind: "pack"` branch is the seam already in place to graft onto.

Sequenced **after** [[ALW-299]] (exposes the base *seams* a pack wires into) and built on [[ALW-327]] D10 layer-sliced layout (done).

## Scope

- A code-bearing **POS** pack under `packages/registry/registry/pos/` following the [[ALW-327]] D10 layer slices: `db / contract / core / server / tools / components` + `skill.md` + `*.test.ts`.
- The **server** slice as a **Hono route** (the starter's Workers stack) — not a Next.js handler.
- Wire `meta.sfabKind: "pack"` through the registry generator + types so a pack item builds into `registry.json` with its full multi-layer file set + `registryDependencies` (today only `block` items are exercised).
- The **ephemeral install skill** (ADR-0017 §2): a one-shot install-time procedure that wires the pack into the base seams, writes the provenance entry, then **deletes itself**.
- The **provenance manifest** (ADR-0017 §3): extend `.sfab/template.json` with `packs: {<name>: {ref, installedAt}}` — **no `mode` flag**; the installed `ref` is the only stored divergence baseline.

## Acceptance criteria

(Carry-over of [[ALW-324]] AC-2…5, now realized through POS.)

- [ ] AC-1: The POS pack is laid out per the D10 layer-sliced layout (`db/contract/core/server/tools/components` + `skill.md`) and resolves to its source files + `registryDependencies` in the built `registry.json`. verify: inspect the built pack JSON.
- [ ] AC-2: `shadcn add sfab-oss/sfab-starter/pos#<ref>` pulls the pack into a fresh project and its install skill wires it. verify: run in a scratch project, or document the dry-run + skill steps.
- [ ] AC-3: Installing the pack writes `packs.pos = {ref, installedAt}` to `.sfab/template.json`, with **no `mode` field**. verify: inspect the manifest after install.
- [ ] AC-4: Layer-1 — the pack is imported/rendered/unit-tested in the registry harness. verify: `pnpm --filter @workspace/registry test`.
- [ ] AC-5: `meta.sfabKind: "pack"` is exercised end-to-end (generator, types, gallery) — no longer a dead branch. verify: a pack item builds and the drift gate stays green.
- [ ] AC-6: README documents the pack model, `#<ref>` coupling, the install-skill + manifest convention, and the public-only constraint. verify: read `packages/registry/README.md`.
- [ ] AC-7: `pnpm typecheck && pnpm lint:check && pnpm test && pnpm build` pass. verify: run all four.

## Non-goals

- The **Layer-2 install-eval harness over sandboxes** (dogfood) — platform-side, separate.
- Promotion intake + multi-template **registry-of-many** — gated on [[ALW-170]].
- Any pack beyond POS; cross-repo bump propagation ([[ALW-306]]).
- A private / custom registry server.

## Boundaries

- ✅ Always: the pack imports from `@workspace/ui` aliases + tailwind tokens; releases on the same tag as the rest of the template (one-tag coupling); keep `sfab-oss/sfab-starter` public; install skills are ephemeral (self-delete) and write the provenance entry.
- ⚠️ Ask first: changing how the template is tagged/released; adding the Layer-2 sandbox harness here.
- 🚫 Never: store a forked/tracked `mode` flag; couple the pack to a vertical's domain naming in the *mechanism*; assume a custom registry server in v1; revert to the flat `{schema, server, components}` layout.
