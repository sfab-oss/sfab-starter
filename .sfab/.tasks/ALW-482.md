# ALW-482 UX layout review — screens, dialogs, empty states (launch cut last)

## Context

After the launch-cut fixes (approval verify, soft-stretch polish), do a focused **UX layout review** of the starter as a stranger would see it: screens, dialogs, empty states, and density — not another full functional QA sweep. File confirmed layout issues as draft fix tasks; do not fix inside this session unless trivial and approved.

Run this **last** under [[ALW-479]], after needed + soft-stretch children land (or are explicitly waived).

## Scope

Walk desktop (and spot-check mobile) across:
- Shell: sidebar, org switcher, breadcrumbs, command palette chrome
- Auth: login / signup (and password-reset if shipped)
- Catalog list + detail; Entities list + detail
- Documents list + composer / detail; Record Payment dialog
- Settings (general + members if routed)
- Org-agent chat: dock, approval card, history, files sheet, common dialogs

Look for: inconsistent spacing/padding, cramped or sparse layouts, dialog width/overflow, misaligned actions, competing hierarchy, broken empty states, mobile sheet issues.

## Acceptance criteria

- [ ] AC-1: Every surface in Scope visited with screenshot evidence in the packet. verify: artifact folder
- [ ] AC-2: Findings logged with severity (block-launch / polish / wontfix) and disposition. verify: tracker markdown
- [ ] AC-3: Block-launch and polish findings filed as draft tasks (or folded into existing ones). verify: [[ALW-###]] links
- [ ] AC-4: Short summary comment on [[ALW-479]] with go/no-go for layout.

## Non-goals

- Functional re-test of money conservation, RBAC matrices, or agent tool correctness (covered elsewhere).
- Visual redesign / new design system.
- Implementing fixes in this task.

## Boundaries

- ✅ Always: seeded local app; evidence per finding.
- ⚠️ Ask first: filing more than ~8 new fix tasks (batch/fold instead).
- 🚫 Never: code changes on this task's branch; prod.
