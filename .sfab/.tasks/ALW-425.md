# ALW-425 QA: catalog end-to-end — ResourceTable, product CRUD, image upload

## Context

Catalog just migrated to the ResourceTable pattern ([[ALW-421]], merged 2026-07-08) and that merge has not been re-verified in a dedicated pass. Product CRUD (create dialog, inline edit, delete confirm, image upload) is the worked example the golden base ships, so it must be exemplary.

## Scope

- List: server pagination, sortable columns, debounced search synced to the URL (deep-link and back/forward behavior), filters shipped with ResourceTable, empty and no-results states.
- Create-product dialog: validation, minor-units price clarity (tracker B-1: no label saying units), dialog padding regression from [[ALW-419]].
- Detail: read view, inline edit round-trip, image upload happy path + oversized/wrong-type file, delete with confirm and post-delete navigation.
- Perf sniff: table behavior with a few hundred seeded products.

## Acceptance criteria

- [ ] AC-1: All list interactions verified on desktop + mobile with evidence, including URL-synced search deep-linking.
- [ ] AC-2: Product create/edit/delete/image-upload round-trips verified with evidence, including at least two invalid-input probes.
- [ ] AC-3: Price minor-units presentation assessed and B-1 resolved into a concrete recommendation or fix task.
- [ ] AC-4: Findings logged and filed as draft tasks.

## Non-goals

- Redesigning the table (ALW-421 just landed; only verify it).

## Boundaries

- ⚠️ Ask first: n/a.
