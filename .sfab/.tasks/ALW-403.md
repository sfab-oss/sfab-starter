# ALW-403 Org-agent: adopt Think `Chats` base class once it ships upstream

## Context

Deferred from [[ALW-398]]. Our multi-session backend is hand-rolled against the shipped `examples/assistant` reference (registry-authoritative chats via `listSubAgents`/`subAgent`/`deleteSubAgent`, `onBeforeSubAgent` 404-gate, parent-owned `Workspace` + child `SharedWorkspace` proxy). Cloudflare's `design/rfc-think-multi-session.md` proposes a library `Chats` base class that would fold `createChat`/`listChats`/`renameChat`/`deleteChat`/`searchChats`/shared-context into the framework, but as of `@cloudflare/think@0.12.1` it is **not shipped** (`packages/think/src/chats.ts` does not exist at origin/main). When it ships, migrate `OrgAgent` onto it to delete our hand-rolled scaffolding.

## Scope

- Track upstream; when `Chats` (and any `RemoteContextProvider`/`useChats`) lands in a released `@cloudflare/think`, replace `OrgAgent`'s hand-rolled chat index + lifecycle RPC with the base class.
- Preserve the current RPC surface consumed by the UI ([[ALW-401]]) or coordinate the change with it.

## Acceptance criteria

- [ ] AC-1: `OrgAgent` extends the shipped `Chats` base class; the hand-rolled `chat_meta` lifecycle scaffolding is removed where the base covers it. verify: diff + green gates + a chat completes a turn.
- [ ] AC-2: The RPC surface ALW-401 depends on is preserved or its migration is coordinated. verify: UI still connects/lists/creates/deletes.

## Non-goals

- Doing this before `Chats` is released (mirror `examples/assistant` until then).

## References

- [[ALW-398]], [[ALW-401]]. Upstream: `cloudflare/agents` `design/rfc-think-multi-session.md` (status: proposed).
