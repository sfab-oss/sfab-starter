# ALW-458 Remove `as unknown as` cast in paused-execution-card type guard

## Context

ALW-456 landed a live `PausedExecutionCard` for mid-script codemode approvals. The output type guard currently ends with `return o as unknown as CodemodeOutput` to satisfy TypeScript after a runtime status/executionId check — a double cast that papers over the narrowing instead of expressing it.

## Scope

- Replace the `as unknown as CodemodeOutput` cast in `apps/web/src/components/chat/tools/paused-execution-card.tsx` with a proper type predicate / discriminated narrowing (or a small validated parse) so the guard returns `CodemodeOutput` without a double cast.
- Keep the same runtime behaviour (paused / completed / rejected / error + `executionId`).

## Acceptance criteria

- [ ] AC-1: `paused-execution-card.tsx` has no `as unknown as` cast (verify: ripgrep the file)
- [ ] AC-2: `asCodemodeOutput` still correctly accepts only outputs with `executionId` + known `status` (verify: existing typecheck / focused unit or workerd coverage if added)
- [ ] AC-3: `pnpm --filter web typecheck` passes

## Non-goals

- Changing approval UX, Think callables, or codemode composition.
- Broad repo-wide cast cleanup beyond this guard.

## Boundaries

- ✅ Always: keep the runtime status/`executionId` checks; prefer a type predicate over casts.
- ⚠️ Ask first: adding a zod schema dependency just for this guard.
- 🚫 Never: weaken the guard to accept arbitrary objects.
