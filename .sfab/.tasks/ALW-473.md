# ALW-473 Spike: migrate sfab-starter to TypeScript 7

## Context

TypeScript 7.0 (Go-native `tsc`) shipped with large typecheck speedups. Both repos are still on TypeScript 5.9.3. This spike evaluates migrating **sfab-starter** first (template for customers), measuring typecheck/build gains and documenting config/tooling friction before considering the same change in `sfab`.

Official path: clear 6.0 deprecations (notably remove `baseUrl`), then adopt 7.0. Starter already uses Biome (no typescript-eslint / Compiler API dependency), so side-by-side `@typescript/typescript6` is likely unnecessary.

## Scope

- Create a branch/worktree and attempt the migration on `sfab-starter` only.
- Remove deprecated `baseUrl` (all current usages are `"."` with already-prefixed `paths` — delete-only).
- Bump TypeScript to 7.x (optionally via 6.0 bridge first) and align workspace pins.
- Run `pnpm typecheck`, `pnpm lint:check`, `pnpm test`, `pnpm build`; record before/after timings.
- Smoke-check tooling that touches TS configs (`drizzle-kit`, vitest, vite/wrangler build).
- Document blockers, config diffs, and a go/no-go recommendation for rolling the same change into `sfab`.

## Acceptance criteria

- [ ] AC-1: Packet worktree exists on branch `alw-473-ts7-starter-spike` from `origin/main` with deps installed.
- [ ] AC-2: Baseline `pnpm typecheck` (and preferably build) timing captured on current 5.9 before the bump. verify: timed command output in packet artifacts
- [ ] AC-3: `baseUrl` removed from all starter `tsconfig*.json` files; `paths` still resolve. verify: `rg baseUrl` empty under tsconfigs; typecheck passes or failures documented
- [ ] AC-4: TypeScript 7.x installed and `pnpm typecheck` run; remaining errors triaged (config vs real). verify: `typescript` version in lockfile + typecheck log
- [ ] AC-5: Full local gates attempted (`typecheck`, `lint:check`, `test`, `build`) with results recorded in PACKET.md / artifacts.
- [ ] AC-6: Written go/no-go note for starter + whether to mirror into `sfab`, including any need for `@typescript/typescript6` side-by-side.

## Non-goals

- Migrating the `sfab` platform repo in this spike.
- Merging to `main` without an explicit human go-ahead after the spike results.
- Changing Biome, Vite, or TanStack major versions unless required to unblock TS 7.
- Editor/LSP rollout policy beyond noting Cursor/VS Code TS 7 server behavior.

## Boundaries

- ✅ Always: work only in the packet worktree; keep canonical starter on `main`.
- ✅ Always: discard wrangler/typegen noise (`worker-configuration.d.ts`, `packages/env/src/env.d.ts`) — do not commit it.
- ⚠️ Ask first: opening a non-draft PR, merging, or expanding into `sfab`.
- 🚫 Never: push to `main`, touch prod, or hand-edit `.sfab/` mirrors.
