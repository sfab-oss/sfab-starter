# ALW-330 sfab-starter: documentation content + ADR cleanup & re-enumeration (knowledge-layer E1‚ÄìE4)

## Context

The docs / knowledge-layer exploration (grill-with-docs, 2026-06-18) locked decisions **E1‚ÄìE4** for how `sfab-starter` structures documentation for BOTH human developers and AI agents. This task **lands the content** and **cleans up the existing, partly-misleading docs**. It is the "content" half that **[[ALW-326]]'s Non-goals explicitly defer** ("the formal `docs/architecture` doc + ADRs are gated by the docs-structure exploration" ‚Äî now complete). ALW-326 scaffolds the empty structure; this task fills it and sets the governing convention. Pairs with the platform task that folds E1‚ÄìE4 into ADR-0010 ¬ßH.

### Locked decisions (E1‚ÄìE4)
- **E1 ‚Äî Keep decision records under a STRICT significance bar.** An ADR is warranted only when a choice is (a) costly/disruptive to reverse, (b) cross-cutting, AND (c) chose among real alternatives. Reject "ADRs for everything." Records capture opinions + reversals (supersede model).
- **E2 ‚Äî Five-bucket taxonomy:** `README.md` (adopter launchpad) ¬∑ `AGENTS.md` (Tier-1 router) ¬∑ `docs/architecture.md` ¬∑ `docs/decisions/` ¬∑ `docs/guides/` (renamed from `template-best-practices/`). **DROP `docs/plans/`** ‚Äî transient plans live in the SFAB task system or Claude Code plan-mode, never `docs/`.
- **E3 ‚Äî `AGENTS.md` is the single Tier-1 source; `.claude/CLAUDE.md` is a byte-identical hand-maintained copy of it** (mirror, NOT a symlink ‚Äî some tools don't follow/auto-read a symlinked instructions file; edit one, copy it over the other; revised 2026-06-18 post-review from an initial symlink decision); tool-specific divergence later (`.cursor/rules`) is its own file. No human/agent doc split. Skills = on-demand Tier-2 (vendored hash-pinned + authored; skills *dirs* stay symlinked), discovery via `description`. Guide-vs-skill boundary. Reference = generated (Tier-3).
- **E4 ‚Äî Starter knowledge lives in-repo `docs/`** (PR-authored, repo is source of truth), NEVER `.sfab/.docs`.
