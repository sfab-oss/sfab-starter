### Locked decisions (E1вҖ“E4)
- **E1 вҖ” Keep decision records under a STRICT significance bar.** An ADR is warranted only when a choice is (a) costly/disruptive to reverse, (b) cross-cutting, AND (c) chose among real alternatives. Reject "ADRs for everything." Records capture opinions + reversals (supersede model).
- **E2 вҖ” Five-bucket taxonomy:** `README.md` (adopter launchpad) В· `AGENTS.md` (Tier-1 router) В· `docs/architecture.md` В· `docs/decisions/` В· `docs/guides/` (renamed from `template-best-practices/`). **DROP `docs/plans/`** вҖ” transient plans live in the SFAB task system or Claude Code plan-mode, never `docs/`.
- **E3 вҖ” `AGENTS.md` is the single Tier-1 source; `CLAUDE.md` is a symlink to it** (identical content, zero drift); tool-specific divergence later (`.cursor/rules`) is its own file. No human/agent doc split. Skills = on-demand Tier-2 (vendored hash-pinned + authored), discovery via `description`. Guide-vs-skill boundary. Reference = generated (Tier-3).
- **E4 вҖ” Starter knowledge lives in-repo `docs/`** (PR-authored, repo is source of truth), NEVER `.sfab/.docs`.
