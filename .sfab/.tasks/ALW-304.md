# ALW-304 Replace Resend with Cloudflare Email in packages/email

## Context

`sfab-starter` sends transactional email (org invitations) via **Resend**, and the template should move to **Cloudflare Email** (Workers `send_email` binding) instead of a third-party API key. Resend + `RESEND_API_KEY` should be removed end-to-end so email works on Cloudflare-native bindings.

**Premise update 2026-07-01 (verified in code):** the original pain — Resend instantiated **at module load**, breaking CI without `RESEND_API_KEY` — has since been fixed: `packages/email/src/index.ts` now lazy-inits via `getResend()` (throws only when `sendMail` is actually called without the key). So CI no longer needs a fake key and the urgency is reduced (priority dropped high → medium). The remaining goal is the platform alignment itself: `resend@^4.1.2` is still a dependency and the send path still requires a third-party API key.

## Scope

- Replace Resend in `packages/email/` with Cloudflare Email sending (Workers `send_email` binding + `EmailMessage` / documented Cloudflare Email Workers outbound API).
- Keep React Email templates (`organization-invitation.tsx`) — render to HTML/MIME for Cloudflare send, or equivalent supported path.
- Update `packages/email/package.json`: remove `resend` dependency.
- Update env/bindings:
  - Remove `RESEND_API_KEY` from `packages/env`, `worker-configuration.d.ts`, `.dev.vars.example`, README env table.
  - Add the Cloudflare email binding in `apps/web/wrangler.jsonc` (+ `wrangler.test.jsonc` if needed for Vitest).
  - Keep `EMAIL_SENDER` and `MOCK_SEND_EMAIL` behaviour where still useful.
- Keep `sendMail` lazy/binding-only so `pnpm test` passes in CI with no email secrets (already true today — preserve it).
- Update README stack table (Resend → Cloudflare Email).
- Auth invitation flow should work unchanged at the call site (`sendMail` API preserved).

## Acceptance criteria

- [ ] AC-1: No `resend` import or `RESEND_API_KEY` reference remains in the repo. verify: `grep -ri resend . --include='*.ts' --include='*.tsx' --include='*.json' --include='*.md' --include='*.jsonc'` returns nothing (lockfile excluded after `pnpm install`).
- [ ] AC-2: `packages/email` sends org-invitation email via the Cloudflare Email binding when configured. verify: dev invite flow or unit/integration test with mock binding.
- [ ] AC-3: `MOCK_SEND_EMAIL=true` still skips real send and logs instead. verify: call `sendMail` with mock flag.

- [ ] AC-5: `pnpm typecheck`, `pnpm lint:check`, and `pnpm build` pass. verify: run all three.

## Non-goals

- Inbound email routing / `email()` handler for receiving mail.
- Porting sfab platform's email package (separate repo task if needed).
- Full production Email Routing DNS setup docs (link to Cloudflare docs only).

## Boundaries

- ✅ Always: preserve the `sendMail(to, templateId, props)` call surface used by `@workspace/auth`.
- ✅ Always: keep React Email templates unless the Cloudflare path requires plain HTML.
- ⚠️ Ask first: changing `EMAIL_SENDER` semantics or adding new required secrets beyond Cloudflare bindings.
- 🚫 Never: reintroduce Resend; require third-party email API keys for the default template path.
