# ALW-304 Replace Resend with Cloudflare Email in packages/email

## Context

`sfab-starter` sends transactional email (org invitations) via **Resend** (`packages/email/src/index.ts` instantiates `new Resend(env.RESEND_API_KEY)` at module load). CI tests fail without `RESEND_API_KEY` in the environment, and the template will move to **Cloudflare Email** (Workers Email / `send_email` binding) instead of a third-party API key.

Resend + `RESEND_API_KEY` should be removed end-to-end so email works on Cloudflare-native bindings and tests/CI no longer require a Resend secret.

## Scope

- Replace Resend in `packages/email/` with Cloudflare Email sending (Workers `send_email` binding + `EmailMessage` / documented Cloudflare Email Workers outbound API).
- Keep React Email templates (`organization-invitation.tsx`) — render to HTML/MIME for Cloudflare send, or equivalent supported path.
- Update `packages/email/package.json`: remove `resend` dependency.
- Update env/bindings:
  - Remove `RESEND_API_KEY` from `packages/cloudflare-env`, `worker-configuration.d.ts`, `.dev.vars.example`, README env table.
  - Add Cloudflare email binding in `apps/web-tanstack/wrangler.jsonc` (+ `wrangler.test.jsonc` if needed for Vitest).
  - Keep `EMAIL_SENDER` and `MOCK_SEND_EMAIL` behaviour where still useful.
- Ensure `sendMail` does **not** throw at import time when secrets are absent (lazy init or binding-only) so `pnpm test` passes in CI without fake Resend keys.
- Update README stack table (Resend → Cloudflare Email).
- Update any auth invitation flow that calls `sendMail` — should work unchanged at the call site if `sendMail` API is preserved.

## Acceptance criteria

- [ ] AC-1: No `resend` import or `RESEND_API_KEY` reference remains in the repo. verify: `grep -ri resend . --include='*.ts' --include='*.tsx' --include='*.json' --include='*.md' --include='*.jsonc'` returns nothing (lockfile excluded after `pnpm install`).
- [ ] AC-2: `packages/email` sends org-invitation email via Cloudflare Email binding when configured. verify: dev invite flow or unit/integration test with mock binding.
- [ ] AC-3: `MOCK_SEND_EMAIL=true` still skips real send and logs instead. verify: call `sendMail` with mock flag.
- [ ] AC-4: `pnpm test` passes in CI without `RESEND_API_KEY` (no module-load Resend constructor). verify: green `test` job on a PR for this change.
- [ ] AC-5: `pnpm typecheck`, `pnpm lint:check`, and `pnpm build` pass. verify: run all three.
