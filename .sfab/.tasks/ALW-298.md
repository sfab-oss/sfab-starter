# ALW-298 Rename project identity from simple-monorepo-starter to sfab-starter

## Context

The repo is linked on the platform as `sfab-oss/sfab-starter`, but the codebase still uses the legacy `simple-monorepo-starter` identity (root `package.json` name, README clone path, turbo cache paths from an old checkout). ADR-0010 and [[ALW-110]] assume the template is canonically named `sfab-starter`. A consistent identity is prerequisite for `.sfab/template.json`, OSS launch ([[ALW-46]]), and downstream de-verticalization work.

## Scope

- Rename root `package.json` `name` to `sfab-starter`.
- Update README setup instructions (`cd sfab-starter`, any other `simple-monorepo-starter` references).
- Update `apps/web-tanstack/package.json` `name` if it still carries legacy naming.
- Update `apps/web-tanstack/wrangler.jsonc` Worker `name` and D1 `database_name` to `sfab-starter` / `sfab-starter-db` (and matching `wrangler.test.jsonc`, db scripts in root + app `package.json`).
- Grep the repo for remaining `simple-monorepo` / `simple-monorepo-tanstack` / `web-tanstack-db` strings and fix user-facing or config references.
- Run `pnpm install`, `pnpm typecheck`, `pnpm lint:check`, `pnpm test`, and `pnpm build` after renames.
- Do **not** add `.sfab/template.json` here — that ships with [[ALW-110]].

## Acceptance criteria

- [ ] AC-1: Root `package.json` name is `sfab-starter`. verify: `node -p "require('./package.json').name"`
- [ ] AC-2: README clone/setup path says `sfab-starter`, not `simple-monorepo-starter`. verify: `grep -n simple-monorepo README.md` returns nothing.
- [ ] AC-3: Wrangler Worker + D1 database names use `sfab-starter` / `sfab-starter-db` consistently (including test config and db scripts). verify: `grep -rn 'web-tanstack-db\|simple-monorepo' apps/web-tanstack wrangler*.jsonc package.json`
- [ ] AC-4: `pnpm typecheck`, `pnpm lint:check`, `pnpm test`, and `pnpm build` pass from repo root. verify: run all four.

## Non-goals

- De-verticalizing or renaming domain entities (inventory → generic) — [[next de-verticalize task]].
- Adding `.sfab/template.json` or platform template linkage — [[ALW-110]].
- Renaming the GitHub repo (already `sfab-oss/sfab-starter`).

## Boundaries

- ✅ Always: PR-only; keep biome formatting; run the full pre-commit check suite.
- ⚠️ Ask first: renaming Cloudflare resources that already exist in a deployed environment (may need manual wrangler/d1 steps).
- 🚫 Never: hand-edit `.sfab/.tasks/*`; change the sfab platform repo.
