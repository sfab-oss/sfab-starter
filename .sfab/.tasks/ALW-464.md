# ALW-464 Bump starter dependencies within minimumReleaseAge

## Context

Starter deps have drifted; we want a routine bump that stays inside the workspace supply-chain gate (`minimumReleaseAge: 2880` / 2 days in `pnpm-workspace.yaml`). pnpm already enforces that on resolve/install, and `pnpm outdated` / `pnpm update` are the supported path â€” no custom tooling needed.

Supersedes the narrow open PR [#24](https://github.com/sfab-oss/sfab-starter/pull/24) (`chore/dep-updates`, hono-only security bump). That PR is closed in favor of one age-gated workspace update PR from this task's `alw-464-starter-deps-age-bump` branch.

## Scope

- Inventory outdated direct deps with `pnpm outdated -r` (age gate applied).
- Apply updates with `pnpm update -r` (in-range) and, where intentional, `pnpm update -r --latest` for selected packages â€” still subject to `minimumReleaseAge`.
- Refresh security `overrides` in `pnpm-workspace.yaml` only when a pin is still required and the mature version moved.
- Land one PR that updates `package.json` files + `pnpm-lock.yaml`; keep `minimumReleaseAge` unchanged unless explicitly decided otherwise.
- Close / leave closed PR #24; do not merge `chore/dep-updates`.
