# ALW-533 Spike: hybrid slash actions on chat-dock input

## Context

Chat composers in the starter are still plain PromptInputTextareas. We want to try a **hybrid** slash-action UX: type `/` → action menu (click + keyboard) → selected actions persist as **badges/chips outside the textarea** (structured state), while the message text stays a normal string. Spike first in the registry **chat-dock** block (bottom-footer chat), not the full-page chat-page block and not the live org-agent dock yet.

## Scope

- Prototype hybrid `/` slash actions in `packages/registry/registry/blocks/chat-dock/` (especially `components/chat-input.tsx` and related demo wiring).
- Keep the existing textarea; do **not** replace with Tiptap/Lexical for this spike.
- On `/`, show a filterable action list with mouse + keyboard selection.
- Selected actions render as dismissible badges/chips (batch tray) beside/above the textarea; submit payload can include `{ text, actions }` (demo/mock is fine).
- Wire enough mock commands to exercise the UX in the registry gallery/docs preview.

## Acceptance criteria

- [ ] AC-1: In the chat-dock registry preview, typing `/` opens an action menu anchored to the composer. verify: open chat-dock block in docs/registry preview
- [ ] AC-2: Actions are selectable via click and keyboard (arrows + Enter/Escape). verify: keyboard-only walkthrough
- [ ] AC-3: Selecting an action adds a dismissible chip/badge outside the textarea and clears/consumes the `/query` trigger text. verify: screenshot of chip tray + remaining message text
- [ ] AC-4: Submitting a message with chips + text demonstrates structured selection (logged or shown in the mock UI). verify: submit in preview
- [ ] AC-5: chat-page / live app dock are unchanged unless shared PromptInput primitives were extended intentionally and remain backward-compatible. verify: no unintended diffs outside chat-dock (+ shared primitives if needed)

## Non-goals

- Full rich-text editor migration (Tiptap/Lexical/Plate).
- Inline mirror-overlay chips inside the textarea.
- Shipping to the live org-agent bottom dock / product routes in this task (follow-up after spike).
- Mentions (`@`) unless they fall out free from the same trigger plumbing.

- 🚫 Never: push to main; hand-edit `.sfab/` task files.
