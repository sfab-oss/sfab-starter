# ALW-533 Migrate chat UI off AI Elements onto shadcn

## Context

Chat surfaces (registry chat-dock / chat-page, and the live org chat dock) still leaned on AI Elements for conversation layout, messages, and the prompt composer. We want those UIs on the starter's shadcn primitives (MessageScroller, Message, Bubble, InputGroup, Empty, compact tokens) with Streamdown for markdown — less one-off AI Elements surface, more shared design-system components.

## Scope

- Replace AI Elements conversation / message / PromptInput usage in chat-dock, chat-page, and live `apps/web` chat with shadcn MessageScroller / Message / Bubble / InputGroup / Empty (and related).
- Render assistant/user markdown via Streamdown directly (no AI Elements MessageResponse wrapper).
- Delete unused AI Elements modules once nothing imports them (`conversation`, `message`, `prompt-input`).
- Keep remaining AI Elements pieces that still earn their keep (e.g. Reasoning, Tool, CodeBlock, ChatVoiceButton) unless a clean shadcn swap already exists.
- Use a small shared chip primitive for composer/message file + page-context affordances (compact, consistent) — not a second design language next to fat attachment cards.

## Acceptance criteria

- [ ] AC-1: Registry chat-dock preview renders messages + composer without AI Elements Conversation / Message / PromptInput. verify: `/view/chat-dock` in docs preview
- [ ] AC-2: Registry chat-page preview uses the same shadcn message/composer stack. verify: `/view/chat-page`
- [ ] AC-3: Live org chat (`apps/web`) uses the same stack for scroller, messages, and composer. verify: typecheck + smoke in dock
- [ ] AC-4: Markdown in chat goes through Streamdown (not AI Elements MessageResponse). verify: assistant reply with markdown in gallery or dock
- [ ] AC-5: Deleted AI Elements files are unused (`conversation.tsx`, `message.tsx`, `prompt-input.tsx`). verify: ripgrep + registry/web typecheck

## Non-goals

- Slash-command / hybrid `/` action menu (out of scope; do not prototype here).
- Replacing Reasoning / Tool / CodeBlock / voice button unless already trivial.
- Broader AI Elements purge outside chat layout/composer.
- Rich-text editors (Tiptap/Lexical).

## Boundaries

- ✅ Always: implement in the packet worktree on `alw-533-…`; keep gallery demos working; prefer existing shadcn primitives.
- ⚠️ Ask first: deleting AI Elements modules still imported outside chat; changing PromptInput APIs for non-chat consumers (should be gone).
- 🚫 Never: push to main; hand-edit `.sfab/` task files; ship slash-command UX in this task.
