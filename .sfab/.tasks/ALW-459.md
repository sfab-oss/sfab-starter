# ALW-459 Adopt shadcn MessageScroller/Message/Bubble/Marker/Attachment for chat (passes 1–3)

## Context

Org chat still uses older `ai-elements` (`Conversation` + `use-stick-to-bottom`, custom message/attachment markup, ad-hoc compaction divider). shadcn shipped a chat kit that matches the Cursor-lite UX we want: turn-anchored scrolling, proper message/bubble layout, attachments, and Marker activity rows.

## Scope

Light passes only (not full Cursor density modes):

1. **Pass 1 — MessageScroller**: replace `Conversation`/`StickToBottom` with MessageScroller; `scrollAnchor` on user messages; jump-to-latest; open at last-anchor.
2. **Pass 2 — Message + Bubble + Attachment**: user text end-aligned Bubble; assistant ghost Bubble; file parts via Attachment/AttachmentGroup in transcript + composer where we already show files.
3. **Pass 3 — Marker**: compaction as separator Marker; settled non-display tools collapse to Marker rows (expand → minimal I/O); in-flight status Markers with shimmer where natural. Keep display tools and Approve/Reject cards expanded.

Add the shadcn components into `@workspace/ui` and wire the web chat.

## Acceptance criteria

- [ ] AC-1: Chat list uses MessageScroller; sending a message anchors the user turn near the top with previous-turn peek (verify: browser)
- [ ] AC-2: User/assistant text and file attachments render via Message/Bubble/Attachment (verify: browser + no leftover StickToBottom Conversation in chat-messages)
- [ ] AC-3: Compaction shows as Marker separator; settled generic/codemode tools collapse to Marker (expand shows minimal I/O); display tools + approval cards stay expanded (verify: browser)
- [ ] AC-4: `pnpm typecheck` / lint / relevant tests pass

## Non-goals

- Codemode inner-step breakdown from execution `log` (pass 4)
- Compact/Balanced/Detailed density toggle
- Full Cursor Agents Window / todo chrome
- Rewriting display tools into canvases

## Boundaries

- ✅ Always: keep Approve/Reject and display-tool cards expanded and actionable; preserve page-context badge and message actions.
- ⚠️ Ask first: deleting `ai-elements` Conversation/Message entirely if other surfaces still import them.
- 🚫 Never: bury mid-script approval inside a collapsed Marker.
