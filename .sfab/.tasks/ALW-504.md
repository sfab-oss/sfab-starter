# ALW-504 Settings: wire destructive confirms (remove/leave, cancel invite, delete org)

## Context

Settings confirm UX audit (2026-07-10) found org settings behind the registry confirm ladder: **Remove** / **Leave** are one-click with toast only; **cancel invitation** is not implemented; **delete organization** has no UI despite better-auth + registry `TypeNameConfirmDialog` prescribing type-to-confirm. Catalog/entity already use `AlertDialog`. See `active/alw-440-settings-sub-routes/artifacts/explorations/settings-confirm-ux.md`.

## Scope

- Wrap **Remove member** and **Leave organization** in an `AlertDialog` (Cancel / confirm) before calling `removeMember`.
- Add **Cancel invitation** on pending invitations with the same T1 AlertDialog pattern + hook.
- Add an owner-only **Danger zone** on `/settings/general` (or adjacent) with **Delete organization** using type-to-confirm (`TypeNameConfirmDialog` from registry pattern — hoist/copy into app shared confirm component as needed).
- Reuse existing catalog/entity AlertDialog patterns; do not invent a third confirm style.

## Acceptance criteria

- [ ] AC-1: Clicking Remove or Leave opens a confirm dialog; Cancel aborts; Confirm runs the mutation. verify: browser.
- [ ] AC-2: Pending invitations show Cancel; confirm dialog required before revoke. verify: browser (seed or invite first).
- [ ] AC-3: Owner sees Delete organization; button stays disabled until org name is typed exactly; successful delete redirects out of the deleted org. verify: browser (use a disposable org if needed).
- [ ] AC-4: Non-owners do not see delete-org control. verify: role check / UI.

## Non-goals

- Unsaved-changes guard on general settings form.
- Reworking invite/accept flows.
- Full registry `settings-page` gallery rewrite (optional small align only if cheap).

## Boundaries

- ✅ Always: match registry confirm ladder (T1 AlertDialog / T2 type-to-confirm).
- ⚠️ Ask first: extra app-side cascade cleanup beyond better-auth delete if required.
- 🚫 Never: change RBAC semantics; never one-click destructive membership actions.
