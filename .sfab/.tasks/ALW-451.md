# ALW-451 Fix local OrgAgent Durable Object name (chat Connection closed)

## Context

ALW-427 QA cannot exercise chat, approvals, tool cards, or workspace files: every send fails with UI **"Couldn't send your message. Connection closed"**. Server logs show PartyServer refusing `this.name` on `OrgAgent` because `ctx.id.name` is unset and there is no `__ps_name` fallback. This blocks the QA-1 approval matrix and most of ALW-427.

Known related upstream: partykit#390 / partyserver 0.5.x + pre-2026-03-15 compat-date alarm/name recovery; local Vite also logs falling back from `compatibility_date` `2026-06-11` to `2026-03-01`.

**🤖 local-qa** · agent · 2026-07-09 21:00

> Root cause confirmed during ALW-427 QA.
> 
> **Why it broke on clean main:** Local Vite/workerd falls back `compatibility_date` to `2026-03-01` (below the 2026-03-15 `ctx.id.name` cutoff). `partyserver@0.5.8` (via `agents@0.17.3`) assumes `ctx.id.name` and no longer bootstraps via `setName()`, so OrgAgent throws on `.name` → UI "Connection closed".
> 
> **Why it worked before:** The canonical `sfab-starter` checkout had *uncommitted* local patches (`getAgentByName` + `x-partykit-room` on `routeAgentRequest`, plus wrangler/vite-plugin pins) that were never landed on `main`. The ALW-422 QA worktree was correctly on clean `origin/main` (`8e7e7a3`) and therefore hit the regression.
> 
> Implementing the server.ts workaround now and verifying chat send.
