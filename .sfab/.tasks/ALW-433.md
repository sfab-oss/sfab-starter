# ALW-433 Entities (customers/suppliers) UI — directory, detail, create/edit + document picker

## Context

Entities are the counterparty primitive of the transaction core, but they have no UI: `hono/org-protected/entities.ts` exposes list/get/create only, and documents attach entities via free-text `entityName` (defaulting to "Walk-in"). Every other gap (wallet, purchase documents, payments views) needs a real entity surface first — this is the foundation task of [[ALW-432]].

## Scope

- New `/entities` section: directory page on the ResourceTable pattern established by [[ALW-421]] (search, pagination, sort), entity detail page, create/edit forms.
- Complete the API for the UI: add update (and archive/deactivate rather than hard delete) endpoints following the existing hono + core patterns; keep contract types in `packages/contract`.
- Entity picker component: searchable select used at document creation (replace free-text-only entry; keep quick "walk-in"/ad-hoc name path), reusable for future purchase flows.
- Sidebar nav entry + command-K page entry; document detail links entity name to the entity page.
- Entity detail shows related documents (by entityId) as a first activity view.

## Acceptance criteria

- [ ] AC-1: Entities appear in sidebar; directory lists seeded entities with search/pagination/sort.
- [ ] AC-2: Create and edit an entity round-trip through the UI; archived entities disappear from the picker but keep history intact.
- [ ] AC-3: Creating an invoice lets me pick an existing entity or type an ad-hoc name; the document stores and displays the linked `entityId` when picked.
- [ ] AC-4: Entity detail shows its documents; document detail links back to the entity.
- [ ] AC-5: Org isolation verified (entities never leak across orgs).
- [ ] AC-6: Browser walkthrough evidence, desktop + mobile.

## Non-goals

- Wallet display on the entity page (that's the wallet task, which builds on this).
- Contact-management depth (multiple addresses/contacts) — keep the base lean.

## Boundaries

- ✅ Always: follow existing layer boundaries (route → hooks → hono → core); org-scoped queries.
- ⚠️ Ask first: any schema migration beyond additive fields needed for archive.
- 🚫 Never: hard-delete entities referenced by documents.
