# ALW-568 Template init: persist Q&A/plan, whole-app transform, eager execution

## Context

Live agent-init audit showed a good interview, but reshape stopped short: nav renames + one new slice, leftover starter language, and phase-gate pauses ("say if I should continue"). Adopters need agents that write product answers + a transform plan into `docs/`, then eagerly reshape the whole product surface — especially AI tools/prompts — pausing only for unclear business context.

## Scope

- Update `docs/template-init.md` only (and AGENTS.md only if a one-line pointer needs wording).
- Require interview answers saved under `docs/` and a separate transform plan file.
- Require whole-app reshape (no hanging starter demo voice); include agent tools/prompts.
- Eager execution: do not stop between phases for permission to continue; stop only for business questions.
- End with a short user-facing tour of what changed / what was kept.

## Acceptance criteria

- [ ] AC-1: template-init tells agents to persist interview Q&A in `docs/` as they go (verify: section + suggested path)
- [ ] AC-2: template-init requires a separate transform plan file covering the whole app, including AI tools/prompts (verify: section)
- [ ] AC-3: guidance says keep/rename the base rather than gut early, but do not leave demo framing hanging after reshape (verify: defaults + not-to-do)
- [ ] AC-4: eager rule — continue unless a business-context question is needed; no "next phase?" gates (verify: explicit bullets)
- [ ] AC-5: after reshape, give a short plain-language tour for the user (verify: path step)

## Non-goals

- Changing app code or adding a new required ritual beyond guidance.
- Teaching agents to delete catalog/documents by default.

## Boundaries

✅ Always: product language for user-facing docs; keep AGENTS.md lean.

🚫 Never: hand-edit `.sfab/`; dump technical forks onto the user.
