# ALW-442 Add email password-reset flow (forgot-password + completion)

## Launch cut (2026-07-10)

**Status: soft stretch** under [[ALW-479]]. In scope for launch polish.

## Context

ALW-424 QA found a dead `/forgot-password` link on login; [[ALW-439]] removed the link because no reset flow exists. Users who forget their password have no recovery path — only signup or admin intervention.

## Scope

- Add `/forgot-password` route with email capture and honest UX (request reset link).
- Wire better-auth password-reset (or equivalent) for email/password accounts.
- Add reset completion route if the auth plugin requires it (e.g. `/reset-password` with token).
- Re-add a "Forgot your password?" link on `login.tsx` pointing to the real route.
- Local dev: document/test behavior without sending real email (log link, mail catcher, or dev-only surface).

## Acceptance criteria

- [ ] AC-1: Login "Forgot your password?" navigates to a working page (no 404). verify: browser QA.
- [ ] AC-2: Submitting a registered email initiates reset; unknown email handled without leaking enumeration (or documented tradeoff). verify: API + UI.
- [ ] AC-3: Completing reset with valid token allows login with new password. verify: end-to-end walkthrough.

## Non-goals

- OAuth-only recovery (GitHub login on platform — out of starter scope).
- SMS / MFA reset.

## Boundaries

- ✅ Always: starter `apps/web` + `packages/auth`.
- ⚠️ Ask first: real email delivery config in shared environments.
- 🚫 Never: prod secrets in repo.
