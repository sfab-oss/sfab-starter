# ALW-492 Hide TanStack Devtools behind URL flag

## Context

The TanStack Devtools FAB (bottom-right) sits on top of the chat dock controls and interferes with browser QA and normal use. We previously tried `triggerHidden` in local WIP but it never landed on main.

## Scope

- Gate `<TanStackDevtools>` with docs-recommended `requireUrlFlag` + `urlFlag: "tanstack-devtools"` in `apps/web/src/routes/__root.tsx`.
- Default local/dev UX: no FAB unless `?tanstack-devtools` is present.

## Acceptance criteria

- [ ] AC-1: Without `?tanstack-devtools`, the "Open TanStack Devtools" control is not present (verify: local `:4011` a11y/snapshot)
- [ ] AC-2: With `?tanstack-devtools`, the Devtools trigger is available
- [ ] AC-3: No other behavior changes

## Non-goals

- Removing the Vite `devtools()` plugin or uninstalling packages.
- Changing chat/attach behavior (ALW-453).

## Boundaries

- ✅ Always: keep hotkey/docs path available via URL flag.
- ⚠️ Ask first: deleting Devtools entirely.
- 🚫 Never: commit secrets; push to main.
