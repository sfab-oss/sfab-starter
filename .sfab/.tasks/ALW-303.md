# ALW-303 Add PR template and SFAB contributor guidance in AGENTS.md

## Context

`sfab-starter`'s `AGENTS.md` only covers Biome and ADRs. The main `sfab` repo documents SFAB platform rules (don't hand-edit `.sfab/.tasks`, MCP-first workflow, PR branch naming for auto-link). Contributors and coding agents working on the starter lack that guidance locally.

A lightweight PR template also helps humans (and sets the shape agents should follow) without adding process weight.

## Scope

- Add `.github/pull_request_template.md` adapted from `sfab` — Summary, linked task id, Changes, How to verify (AC ids), Evidence, Risk/rollback. Keep it short.
- Expand `AGENTS.md` with a **SFAB-managed content** section mirroring sfab essentials:
  - `.sfab/.tasks/` and `.sfab/.docs/` are platform-synced; don't hand-edit platform-owned paths.
  - Use SFAB MCP for tasks/docs.
  - PR branch naming: `{org-slug}-{number}-description` (e.g. `alw-298-rename-to-sfab-starter`).
- Do **not** duplicate the full sfab AGENTS.md (no wrangler tail, no engine internals).

## Acceptance criteria

- [ ] AC-1: `.github/pull_request_template.md` exists with Summary + AC verification sections. verify: file present.
- [ ] AC-2: `AGENTS.md` documents `.sfab` sync rules and PR branch naming. verify: read file.
- [ ] AC-3: No instructions contradict the existing `.husky/pre-commit` `.sfab` guard. verify: cross-check hook + AGENTS.md.

## Non-goals

- Full sfab AGENTS.md port.
- MCP server setup docs (link to sfab's `MCP-SERVER.md` if needed, don't copy the whole doc).
- Changing commit message conventions.

## Boundaries

- ✅ Always: keep docs concise; template is guidance, not enforced by CI.
- 🚫 Never: hand-edit `.sfab/.tasks/*` in this PR.
