# ALW-335 Find and remove unused functions, types, and dead code

## Context

`sfab-starter` was extracted and evolved quickly from `sfab` and early scaffolding. That left behind unused exports, dead helper functions, orphaned types, and imports that nothing references. Dead code increases cognitive load for humans and agents, slows typecheck/lint, and makes refactors riskier because it is unclear what is still live.

This is a companion to [[ALW-334]] (comment cleanup) but targets **executable surface area** — functions, types, constants, and modules that are no longer referenced.

## Scope

- Find and remove unused functions, methods, and private helpers that have no callers in the repo.
- Remove unused exports (re-export barrels, package `index.ts` entries) when the underlying symbol is unreferenced.
- Remove unused types/interfaces/type aliases and unused constants when nothing imports or references them.
- Remove orphaned files/modules that are not imported anywhere (after confirming they are not entrypoints, config, or test fixtures).
- Clean up imports left dangling after removals.
- Prefer static analysis (`pnpm typecheck`, Biome unused rules, `knip` or equivalent if already in repo) over guesswork; confirm each removal is safe.

## Acceptance criteria

- [ ] AC-1: No unused exported functions or types remain in `apps/` and `packages/` that static analysis flags as unreferenced. verify: run the repo's dead-code/unused-export check (or add a one-off `knip`/`ts-prune` pass documented in the PR) and resolve all findings in scope
- [ ] AC-2: Orphaned source files with zero importers are removed or justified in the PR (entrypoints, Workers bindings, test helpers, and config excluded). verify: importer graph / tool report shows no unjustified orphans
- [ ] AC-3: Barrel `index.ts` re-exports match actual usage — no dead re-exports. verify: spot-check `packages/*/src/index.ts` and root package exports
- [ ] AC-4: `pnpm lint`, `pnpm typecheck`, and `pnpm test` pass with no behaviour changes. verify: run all three from repo root
- [ ] AC-5: Anything kept despite looking unused has a one-line rationale in the PR (dynamic import, framework convention, planned public API). verify: PR description lists borderline keeps

## Non-goals

- Removing code that is part of the intentional public API surface for template consumers (document in PR if uncertain).
- Performance refactors or renaming for clarity — deletion of dead code only.
- Adding a permanent CI dead-code gate (optional follow-up).
- Touching `.sfab/` mirror files by hand.
- Removing commented-out code (covered by [[ALW-334]]).

## Boundaries

- ✅ Always: prove unreachability before deleting; run lint/typecheck/test; prefer small focused commits/PR.
- ⚠️ Ask first: deleting symbols that might be consumed by fabricate/template hooks, Workers runtime reflection, or undocumented dynamic imports.
- 🚫 Never: remove test-only helpers still used by tests; strip generated/vendor output; change runtime behaviour of live paths.
