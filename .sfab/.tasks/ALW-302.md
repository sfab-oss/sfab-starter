# ALW-302 Raise Node heap in pre-commit hook to prevent build OOM

## Context

The `sfab` pre-commit hook sets `NODE_OPTIONS=--max-old-space-size=8192` before `pnpm build` because the platform Vite build can OOM under the default Node heap (ALW-127). `sfab-starter`'s pre-commit runs `pnpm build` without a heap bump — the starter build is smaller today but already heavy (~15s+), and will grow as features land.

Aligning pre-commit with sfab avoids flaky local commits on memory-constrained machines and agents.

## Scope

- Add `export NODE_OPTIONS="--max-old-space-size=8192"` (or `4096` if sufficient after testing) at the top of `.husky/pre-commit`, before `pnpm build`.
- Keep the existing `.sfab` platform-owned path guard and the lint-staged → typecheck → build → test sequence.
- Optionally narrow `lint-staged` in `package.json` from `"*"` to `*.{js,ts,jsx,tsx,json,css}` to match sfab (only if it doesn't miss files Biome should format).

## Acceptance criteria

- [ ] AC-1: `.husky/pre-commit` exports `NODE_OPTIONS` before `pnpm build`. verify: `grep NODE_OPTIONS .husky/pre-commit`.
- [ ] AC-2: A full commit (or running the hook commands manually) completes `pnpm build` without heap OOM on a typical dev machine. verify: commit or run hook steps locally.
- [ ] AC-3: Existing pre-commit steps still run in order: `.sfab` guard → lint-staged → typecheck → build → test. verify: read hook file.

## Non-goals

- Changing which commands run in pre-commit (still the full suite).
- CI workflow heap settings — covered in the CI task (build job gets `4096`).

## Boundaries

- ✅ Always: minimal diff to `.husky/pre-commit` (+ optional lint-staged scope tweak).
- 🚫 Never: remove the `.sfab` platform-owned path guard.
