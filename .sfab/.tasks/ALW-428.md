# ALW-428 QA: cross-cutting UX — theming, states, responsive, a11y, i18n

## Context

Horizontal qualities that no per-area session owns: dark/light theming, loading/empty/error states, toast consistency, responsive behavior between phone and desktop, keyboard/accessibility basics, and copy consistency. Known seeds: a hardcoded Spanish toast in `invite-member-form.tsx` inside an otherwise-English app; the protected-shell error boundary; Today page is an intentional placeholder (assess presentation, don't invent metrics).

## Scope

- Theming: every page + dialogs + chat in dark and light; contrast of badges/buttons (tracker noted Reject-button contrast).
- States: for each page — loading skeletons, empty (fresh org with no data), error (kill API mid-session, hit the error boundary), toast tone/wording consistency.
- Responsive: tablet-width band (between mobile sheet and desktop rail), text truncation, wide-content overflow.
- A11y basics: keyboard-only walk of the main flows, focus trap + return in dialogs, sr-only labels present, focus-visible styles.
- Copy: language consistency (the Spanish toast), title-casing, terminology drift (Home vs Today).

## Acceptance criteria

- [ ] AC-1: Dark + light sweep of all pages and major overlays with evidence.
- [ ] AC-2: State matrix (page × loading/empty/error) documented with gaps flagged.
- [ ] AC-3: Keyboard-only pass of login→catalog→invoice→payment→chat documented with obstacles listed.
- [ ] AC-4: Spanish toast and any other locale drift filed.
- [ ] AC-5: Findings logged and filed as draft tasks.

## Non-goals

- A full WCAG audit; building Today-page metrics.

## Boundaries

- ✅ Always: fresh org for empty states; evidence per finding.

- 🚫 Never: code changes on this branch.
