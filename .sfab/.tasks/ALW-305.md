# ALW-305 Port sfab vitest-pool-workers / workerd test harness (ALW-241 pattern)

## Context

The sfab platform repo adopted a three-project Vitest setup in June 2026 (ALW-241) using `@cloudflare/vitest-pool-workers` to run real workerd tests for Durable Objects and Workflows. sfab-starter already uses an older pool-workers setup (`^0.13.0`) for D1/API integration tests but lacks the multi-project split, dedicated fixture worker, `.workerd.test.ts` convention, and DO-specific test patterns.

This is prerequisite infrastructure for Tier-2 in-workerd tests once Think/OrgAgent DOs land ([[ALW-218]]). Not blocking initial OrgAgent implementation — node mocks suffice for Tier-1.

Reference: sfab `apps/platform/vitest.config.ts`, `wrangler.test.jsonc`, `src/workerd-test/`, task ALW-241.

## Scope

- Bump `@cloudflare/vitest-pool-workers` to match sfab (`^0.16.15` or latest compatible) and align wrangler.
- Split Vitest into projects: `node` (unit/API), `workers` (`*.workerd.test.ts` DO tests), optional `workers-workflows` if Workflows are added later.
- Add minimal `src/workerd-test/worker-entry.ts` + updated `wrangler.test.jsonc` with test DO bindings (separate from prod TanStack Start bundle).
- Add `workerd-test-env.d.ts` + `src/workerd-test/test-env.ts` for fixture-only binding types.
- Adopt `*.workerd.test.ts` naming convention.
- Update `test` script: `vitest run --project node --project workers`.
- Add reference DO test (TestCounter-equivalent) proving `runInDurableObject` works.
- CI: required node + workers; optional on-demand workflows project if added.

## Acceptance criteria

- [ ] AC-1: `@cloudflare/vitest-pool-workers` bumped and `pnpm install` succeeds. verify: `grep vitest-pool-workers apps/web-tanstack/package.json`
- [ ] AC-2: Vitest config defines `node` and `workers` projects; existing integration tests still pass under appropriate project. verify: `pnpm test` from monorepo root.
- [ ] AC-3: `wrangler.test.jsonc` + `src/workerd-test/worker-entry.ts` exist with at least one test DO binding. verify: `ls apps/web-tanstack/src/workerd-test/`
- [ ] AC-4: At least one `*.workerd.test.ts` passes using `runInDurableObject`. verify: `pnpm test` includes workerd project green.
- [ ] AC-5: `workerd-test-env.d.ts` included in tsconfig types. verify: `pnpm typecheck` passes.

## Non-goals

- Porting sfab Workflow introspection suite (TaskRunWorkflow, merge gates) — only if sfab-starter adopts Workflows.
- Tier-2 workerd tests for Think/OrgAgent DOs — follow-on after [[ALW-218]] DO classes exist.
- Changing production wrangler bindings beyond what OrgAgent needs.

## Boundaries

- ✅ Always: base harness on sfab ALW-241 reference implementation.
- ✅ Always: keep existing D1/API integration tests passing.
- ⚠️ Ask first: if restructuring test directory layout breaks CI or e2e.
- 🚫 Never: change the sfab platform repo.
