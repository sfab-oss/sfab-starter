# ALW-673 Starter: decide ideal logging convention (sharpen later)

## Context

Platform landed a structured logging helper + convention ([[ALW-662]]) and Biome `noConsole` ([[ALW-671]]). Starter has neither yet. Before copying platform wholesale (or inventing something else), decide what logging shape is ideal for **starter apps** (TanStack Start / Workers templates that customers fork).

Sharpen when picked up — this ticket is a placeholder for that interview.

## Scope (draft)

- Inventory how starter apps log today (raw `console.*`, any helpers, Workers Logs expectations).
- Compare options: adopt platform `@sfab/log`-style package (copy/adapt), thinner starter-local helper, or stay console-only with lighter gates.
- Recommend envelope/`kind` vocabulary depth appropriate for template code (not platform-scale ops).
- Note implications for [[ALW-672]] (`noConsole` half of smell gates).

## Acceptance criteria

- [ ] AC-1: Written recommendation (note or task update) on the preferred starter logging approach and why. verify: doc or sharpened task body
- [ ] AC-2: Explicit follow-up tasks filed (or ALW-672 updated) for implementation vs “not needed”. verify: linked tasks

## Non-goals

- Implementing the helper in this task (unless sharpening collapses to a tiny PR).
- Changing platform `@sfab/log`.
