# ALW-327 Directory / package structure (Tier-1 foundation)

## Context

Tier-1's **first** task: decide the monorepo **directory/package structure** before anything is scaffolded or built (ADR-0010 §E). It informs the scaffold ([[ALW-326]]), which implements it. This is **design + decision**. The Transaction Core hub design is the **separate** follow-on [[ALW-328]], done *after* the scaffold.

Order: **this (structure) → [[ALW-326]] (scaffold) → [[ALW-328]] (Transaction Core).**

**Design status: COMPLETE.** Method = `grill-with-docs`, grounded in 4 studied repos (tanstarter-plus `bf3f242`, supermemory `313a8df`, midday `5158731`, dub `50d17cb`) + our `sfab`/`sfab-starter`. ## Outcome — decisions (D1–D10)

- **D1 — Layer-sliced, feature-keyed.** A fixed small set of layers; a capability = the `<cap>` slice present in each layer; a pack adds one more `<cap>`. Rejected package-per-domain (midday) and the fat single `core` bucket (sfab today).
- **D2 — Two independent schema/type sources.** `db` (drizzle) leaf → `$infer` row types; `contract` leaf (pure boundary zod) → `z.infer` types; **no drizzle→zod derivation**; drift caught by per-capability round-trip tests, not the compiler. Retires the `types` and `primitives` packages.
- **D3 — Direction-split types.** Inbound (inputs/commands) = explicit zod in `contract`, shared as runtime **values** by API/MCP/AI-tools + client forms; `core` fn params = `z.infer<contract>` (no hand-sync). Outbound (reads) = **inferred** from the implementation via the typed Hono client (`InferResponseType`); server uses `core`'s `Awaited<ReturnType>`. **Timestamps = ISO `text` end-to-end.** *(Money was specified here as a numeric→number customType; that decision was later **reversed** by Transaction Core **ADR-006** to **integer minor units** — the migration is owned by [[ALW-299]].)* Identification by import location + suffix; no `Db*` prefix.
- **D4 — Domain in a fixed package set; queries in `core`; `db` a leaf.** Not in-app (we're multi-surface — `core` must be framework-agnostic), not package-per-domain.
- **D5 — `db` singleton via `import { env } from "cloudflare:workers"`; one schema file per capability.** `core` is Workers-runtime code; testable via the workerd test pool (alw-241). Affirms + extends ADR-004. Migrations in `packages/db/drizzle/`.
- **D6 — One Worker app `apps/web` hosts all surfaces** (UI + Hono API organized auth-scope→feature + MCP + AI Durable Objects + jobs). New app only when the runtime genuinely differs. **Naming rule = role over technology:** `web-tanstack`→`web`, `db-d1`→`db`, `types`→`contract`, `cloudflare-env`→`env`, `typescript-config`→`tsconfig`; workspace scope stays generic `@workspace/*`.
- **D7 — `ui` = primitives/design-system only**; feature/composite components in-app; **cross-layer feature-key consistency** (`<cap>` identical across layers — `db` one file, other layers a folder). The UI **composition idiom is a DEFERRED exploration** (Radix→Base UI; seed `components.build/llms.txt`).
- **D8 — Dependency boundaries mechanical-by-construction** (the `package.json` dep-graph + the `cloudflare:workers`/server-only import guard) + a CI cycle check; skip tsconfig project-refs and a custom ESLint boundary plugin. **Drop Ultracite; self-maintain Biome rules** (concrete ruleset = a deferred task).
- **D9 — Cross-cutting homes:** auth=`packages/auth`; typed env=`packages/env`; AI=`packages/agent` (the one *packaged* surface, `tools/<cap>.ts` + DO classes bound in `apps/web`); system logging thin (Workers console + tail); **domain activity/audit log = a capability slice (to build)**; jobs logic in `core` + handlers in `apps/web`; **i18n EN+ES (to build, library deferred)**; knowledge layer = root `docs/` + `.agents/skills` (+ `.claude`/`.agent` symlinks) + `AGENTS.md`.
- **D10 — Packs authored vertical, installed sliced.** `registry/<pack>/{db, contract, core, server, tools, components, skill.md, *.test.ts}`; `registry.json` maps each pack file to its layer `target`; `apps/registry` = showcase + Layer-1 harness. Refines ADR-0017 Decision 6.

**A full capability slice spans:** `db/schema/<cap>.ts` · `contract/<cap>/` · `core/<cap>/` · `apps/web/src/hono/<auth-scope>/<cap>/` · `packages/agent/tools/<cap>.ts` · `apps/web/src/components/<cap>/`. A pack drops one `<cap>` slice into each.
