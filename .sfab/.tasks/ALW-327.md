# ALW-327 Directory / package structure (Tier-1 foundation)

## Context

Tier-1's **first** task: decide the monorepo **directory/package structure** before anything is scaffolded or built (ADR-0010 §E). It informs the scaffold ([[ALW-326]]), which implements it. This is **design + decision**; the output is a documented structure. The Transaction Core hub design (and the primitives around it) is the **separate** follow-on task [[ALW-328]], done *after* the scaffold.

Order: **this (structure) → [[ALW-326]] (scaffold) → [[ALW-328]] (Transaction Core).**

## Scope

- Map every Tier-1 primitive (Identity, i18n, Entity & Contact, Catalog, Transaction Core, File Hub, Activity & Audit Log, Reporting, AI Agent, jobs, Notifications) to its home: `packages/` vs `apps/`, and within that where schema / services-core / types / UI / routes live.
- Place the special apps: the web app, `apps/docs` (renderer), `apps/registry` (showcase + build, ADR-0017 §6).
- Naming + package-boundary conventions (domain-neutral; shared-package vs app-local; how `@workspace/*` aliases are organized).
- **Output:** a `docs/architecture/` structure doc (and/or an ADR) that the scaffold implements.

## Acceptance criteria

- [ ] AC-1: A directory/package-structure doc maps every Tier-1 primitive to its home (packages/apps; schema/core/UI/routes) plus the web/docs/registry apps. verify: read the structure doc.
- [ ] AC-2: Naming + package-boundary conventions are documented (domain-neutral; shared-vs-app-local; `@workspace/*` layout). verify: read the doc.
- [ ] AC-3: The structure is concrete enough that [[ALW-326]] can lay out the skeleton 1:1 from it. verify: ALW-326's Tier-1 folders map to this doc.

## Non-goals

- Transaction Core / primitive architecture — [[ALW-328]].
- **Implementing** the structure (laying out the folders/stubs) — that's the scaffold [[ALW-326]].
- The registry / pack mechanism — [[ALW-324]] / ADR-0017.
- Generalizing the inventory example — [[ALW-299]] (this informs its target naming).

## Boundaries

- ✅ Always: domain-neutral naming; design to the ADR-0010 §E parts list; *legible, not optimal* (§D stopping line).
- ⚠️ Ask first: introducing a new top-level package/app convention that diverges from the current monorepo.
- 🚫 Never: couple the structure to a vertical/country; over-engineer.
