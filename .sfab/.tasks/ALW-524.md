# ALW-524 Org-agent tools: ToolResult fail contract + authoring guide

## Context

Org-agent tools today throw (`DomainError`, RBAC) or return silent `null`/`undefined` on miss. Throws become codemode `status: "error"` / Error badges; the desired contract is: **the toolcall succeeds with a structured fail message** so the model can react. Pre-launch: also delete the empty deprecated `getOrgAgentApprovalTools` stub (see [[ALW-523]] for a broader deprecated scrub — this task covers the approval stub as part of the tools pass).

## Scope

- Add a shared `ToolResult<T>` union + `asToolResult` / thin `defineOrgTool` helper under `packages/agent`.
- Migrate all ERP tools in `getOrgAgentTools` / read-only set to return `ToolResult` (map `DomainError` + permission denied; no throw for domain misses).
- Remove `getOrgAgentApprovalTools` stub + update tests/docs that reference it.
- Keep `needsApproval` on `delete_product` (codemode pause unchanged).
- Optional: `toModelOutput` for `ok: false` → model-facing error text.
- Update system prompt briefly for the Result shape.
- **Write/update** `docs/guides/` authoring guide (extend `agent-tool-approvals.md` or sibling): layout, context vs params, needsApproval↔codemode, which AI SDK fields reach sandbox, Result contract, gotchas.

## Acceptance criteria

- [ ] AC-1: Shared `ToolResult` + wrapper used by ERP tools; domain miss/RBAC return `{ ok: false, … }` without throwing
- [ ] AC-2: `delete_product` still pauses via `needsApproval` / codemode approval; after approve, not-found is soft Result (not silent success)
- [ ] AC-3: `getOrgAgentApprovalTools` removed from package exports; tests/docs updated
- [ ] AC-4: Authoring guide updated with Result contract, needsApproval↔codemode, sandbox field gotchas
- [ ] AC-5: Unit/workerd tests cover Result success + fail paths for at least product delete/update/get

## Non-goals

- Full ALW-523 deprecated inventory across the monorepo.
- Exposing money/document mutation tools.
- AI SDK 7 / Think upgrade.
- Redesigning display tools beyond Result consistency if they stay top-level echoes.

## Boundaries

- ✅ Always: toolcall succeeds with structured Result for domain failures; document the contract.
- ⚠️ Ask first: changing public Result field names after this lands.
- 🚫 Never: reintroduce empty deprecated approval stubs.
