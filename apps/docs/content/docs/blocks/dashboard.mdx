The operator dashboard composes the starter's real layout primitives —
`AppLayout`, `Sidebar`, `DataTable`, and `Card` — into the same shape as the
`apps/web` protected shell, but standalone and on mock data.

<BlockViewer name="dashboard-01" />

## What it demonstrates

- **App shell** — `AppLayout` + a collapsible icon `Sidebar` with active nav state.
- **Header** — breadcrumbs, page title, and a primary action.
- **Stat cards** — the four-up KPI row from the dashboard pattern.
- **Data table** — the brand `DataTable` (client-side filter, sort, paging, column visibility) over mock inventory.
- **Activity feed** — a compact event list.
