Full list page block: `Shell` sidebar frame, composed page header, and a full-bleed filter
toolbar + `ResourceTable` inside the inset panel. Data comes from a React Query
hook (`usePeopleList`) over 28 mock people with server-style pagination (10 per
page), sorting, and filters — swap `fetchPeopleList` for a Hono route when you
wire up the API.

<BlockViewer name="resource-list-page" />

## What it demonstrates

- **Shell** — `Shell` + `ShellPage` + `ShellHeader` + `ShellContent` (compound layout).
- **Table kit** — filter toolbar, sort control, sortable headers, row actions.
- **Data layer** — `lib/people/` mock data + `fetchPeopleList`, `hooks/use-people.ts`
  (`peopleListQueryOptions` / `usePeopleList`).
- **Collection states** — skeleton (initial load), stale banner (background refetch),
  preset-empty (filters with no matches + clear CTA), true empty (no records + create CTA),
  and error + retry. All five react only to the `usePeopleList` result, so the page and
  components are production-grade as-is. Flip between them with the **draggable preview
  dock** on [`/view/resource-list-page`](/view/resource-list-page) — drag its handle to
  snap it to any corner.
